## Summary

- 点击移动后，玩家是否能立刻理解“当前位置已变化”，而不是误以为移动按钮锁死。

## Jam 回收口径

本轮优先使用 Jam 录屏回收体验问题。录屏不需要追求完整通关，只要覆盖“点击前预期、点击后反馈、是否误解”即可。

建议每次录屏至少包含：

- 新开局后的前三次点击。
- 一次移动到新节点后的界面停留。
- 一次禁用行动或隐藏路线出现前后的对比。
- 一次遭遇决断或药堂恢复后的反馈阅读。

回收时重点看：

- 截图中是否有足够醒目的当前位置变化。
- 最近反馈是否比因果账更先被看到。
- 状态条里的空窍、AP、地点、暴露、债务是否能支撑下一步判断。
- 玩家是否需要翻页才能确认刚刚的点击成功。

## Handling Rules

- `阻断` 和 `高误导` 必须在 Sprint 2.3 内修复。
- `体验弱` 可以做低风险 UI 或文案微调；如果会牵动规则、内容节奏或剧情结构，则延期到 Sprint 3。
- `文本量少 / 剧情推进感弱` 只登记为 Sprint 3 内容问题，本轮不扩写。
- 如果只改 UI 派生呈现，不升级 `RULES_VERSION`。
- 如果改 AP、元石、债务、暴露、伤势、遭遇代价或修行门槛，必须升级 `RULES_VERSION` 并补测试。
- 如果改 `content/s0` 文本或内容数据，必须升级 `ContentManifest.version`。

## Exit Criteria

Sprint 2.3 只有满足以下条件，才能冻结为“真人走查通过候选”：

- 4 条路径至少完成一次真人点击记录，或提供可审阅的 Jam 录屏。
- 没有未处理的 `阻断` 或 `高误导`。
- 玩家能在移动后不翻因果账也理解当前位置变化。
- 玩家能在开局理解空窍已开、根基未稳、可以开始修行。
- `.\verify-game.cmd` 通过。
- runtime AI 仍不进入 `resolve_action`、Tauri command 或 UI 点击主链。
