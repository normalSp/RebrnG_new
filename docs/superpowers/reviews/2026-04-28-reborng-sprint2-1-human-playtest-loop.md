## 四条必走路径
每条路径都从“新开一局”开始，不复用上条路径的存档状态。走查时不要急着证明能通关，优先记录第一次困惑、第一次想点却不能点、第一次不理解代价的位置。

| 路径 | 必须确认 |
| --- | --- |
| 月光根基 | 能理解 AP、元石消耗、月光修行痕迹、学堂压力、阶段收口。 |
| 黑市跑路 | 能理解黑市不是开局商店，必须先有门路；深夜进入后遭遇勒索，跑路有价值。 |
| 药堂债务恢复 | 能理解重伤不会立刻死亡，药堂恢复会转成债务和人情压力。 |
| 传承诱惑撤退 | 能理解传承是半真半假、高风险诱惑，不是稳定奖励路线。 |

## 问题分级
记录问题时只用四档：

- `阻断`：无法启动、按钮不可用、路径无法继续、关键状态丢失。
- `误导`：UI 或文案让玩家误解系统，例如把黑市看成普通商店、把传承看成稳定收益。
- `体验弱`：能玩但阅读负担高、行动层级不清、压强体感不稳。
- `文案弱`：语气不够冷峻、短标签不够清楚、反馈不像账本批注。

处理规则：
- 有 `阻断` 或 `高误导`：进入 Sprint 2.1 缺陷回收，只做最小修复。
- 只有 `体验弱 / 文案弱`：先汇总，不急着修规则。
- 修改 AP、元石、债务、暴露、伤势或遭遇代价时，必须升级 `RULES_VERSION` 并补测试。
- 只改 UI 文案或布局时，不升级规则版本。

## 验收标准
Sprint 2.1 可以冻结为“真人走查通过候选”的条件：

- 4 条路径都能从新开局走到阶段收口。
- 没有 `阻断` 或 `高误导` 未处理。
- 玩家能用自己的话说清：AP、元石、暴露、债务、伤势、黑市门路、传承风险。
- `.\verify-game.cmd` 仍全量通过。
- runtime AI 仍不进入 `resolve_action`、Tauri command 或 UI 点击主链。
