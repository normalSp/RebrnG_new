# RebrnG 原著世界观与设定一致性总收口审阅

状态：review draft

审阅日期：2026-04-26

审阅对象：`docs/superpowers/specs/` 下 16 份规格文档

审阅目的：在不改动现有规格正文的前提下，判断当前方案是否存在与《蛊真人》世界观、修为逻辑、资源逻辑和剧情锚点的明显冲突，并找出会影响正式架构设计的高风险偏离与关键缺漏。

## 审阅范围与方法

本轮只做只读审阅，不回改规格正文。

审阅覆盖以下五类问题：

- 首发闭环冲突：时间 / AP、地图、经济、遭遇、Build、UI 是否互相打架
- 长期兼容冲突：升仙、仙窍、宝黄天、杀招、阵、仙蛊屋是否要求推翻首发底座
- 原著硬冲突：是否有条目明显违背《蛊真人》核心世界规则
- 软性设定风险：是否有条目过度现代化、平台化、游戏化，容易漂出原著质感
- 关键缺漏：哪些后续架构前仍应先保留成一等状态

本轮审阅同时对照了当前规格内部交叉关系，并额外核对了几项原著常见设定要点：

- 升仙与蛊仙阶段的“三气 / 灾劫 / 福地-洞天”语义
- 宝黄天的接入与交易方式
- 仙窍的经营压力与时间性
- 流派、道痕、蛊虫养用之间的长期关系

## 审阅结论

- `P0 硬冲突`：未发现
- `P1 高风险偏离`：2 项
- `P2 关键缺漏`：3 项
- `P3 建议补强`：2 项

总体判断：**仍可进架构，但不建议把当前文本直接当作“原著一致性已封板”的最终设计基线。**

更准确地说，当前规格已经足够支撑正式架构的对象分层与模块边界设计；但在升仙、仙窍、宝黄天和高阶 Build 这四块上，仍有几处原著关键机制没有被保留成一等状态。它们现在还不是“写错了”，但如果继续空着，后面很容易在实现中被顺手写偏。

## Findings

### P1 高风险偏离

#### 1. `Build` 的“路径倾向”仍混写了求活路线与修炼流派，后期容易撞上原著的流派 / 道痕语义

定位：

- `docs/superpowers/specs/2026-04-26-reborng-routes-build-formation-spec.md:58-81`
- `docs/superpowers/specs/2026-04-26-reborng-longterm-extension-state-boundary-spec.md:156-195`
- `docs/superpowers/specs/2026-04-26-reborng-architecture-input-checklist.md:37-53`

判断：

- 当前写法对首发是成立的，因为首发只需要表达“你更像怎么活”。
- 但 `路径倾向` 下面同时放了 `光道倾向 / 力道倾向` 和 `黑市投机倾向 / 传承跳跃倾向`，这实际上把“修炼主修”与“生存策略”塞进了一个层。
- 在《蛊真人》里，流派、道痕、主修与兼修成本，和黑市偏门、势力依附、求活风格，不是同一种东西。

风险：

- 如果架构直接把这层固化成一个对象，后续做升仙、道痕积累、仙蛊屋组合和杀招主轴时，会出现一个字段同时承担“流派”和“活法”的双重语义。
- 这不一定立刻违背原著，但很容易把后续系统写成“世界观上不够像《蛊真人》”。

建议处理：

- 后续回改时，把 `Build` 里的“求活路线层”和“主修流派层”拆成明确的两层。
- 首发仍可保留现在的玩法体验，但架构对象上不要把二者合并成一个长期字段。

#### 2. 宝黄天目前更像“高价值挂单平台 + 信用空间”，有现代化交易平台漂移风险

定位：

- `docs/superpowers/specs/2026-04-26-reborng-immortal-trade-treasure-yellow-heaven-spec.md:37-45`
- `docs/superpowers/specs/2026-04-26-reborng-immortal-trade-treasure-yellow-heaven-spec.md:61-68`
- `docs/superpowers/specs/2026-04-26-reborng-immortal-trade-treasure-yellow-heaven-spec.md:127-136`

判断：

- 现文档已经正确避开了“宝黄天 = 大号黑市”的错误，这一点是对的。
- 但当前定义把重点压在 `信用空间 / 挂单 / 长期供需` 上，仍偏像现代平台经济抽象。
- 《蛊真人》里的宝黄天更强的辨识度，不只是“信用”，还包括接入媒介、交易方式、特殊空间属性和高价值资源流通机制。

风险：

- 如果后续实现只围绕“卖家可靠度”和“挂单系统”建模，容易把宝黄天写成一个高级拍卖行或电商页。
- 这会削弱原著里那种“特殊空间、特殊媒介、特殊门槛”的异质感。

建议处理：

- 后续回改时，把 `信用` 从“第一主语义”下调为“交易后果层”。
- 需要补出一层更贴原著的接入和结算语义，例如：接入媒介、交易能见度、价值显露方式、特殊空间门槛。

### P2 关键缺漏

#### 3. 升仙规格没有把“三气、灾劫、福地品级”保留成不可跳过的硬钩子

定位：

- `docs/superpowers/specs/2026-04-26-reborng-cultivation-stage-advancement-ascension-spec.md:99-107`
- `docs/superpowers/specs/2026-04-26-reborng-cultivation-stage-advancement-ascension-spec.md:165-188`
- `docs/superpowers/specs/2026-04-26-reborng-cultivation-stage-advancement-ascension-spec.md:190-210`

判断：

- 这份规格成功定义了“升仙是结构断层”，这是对的。
- 但它现在只冻结了“修为准备 / 资源准备 / 结构准备”三类抽象条件，没有把原著中最关键的升仙硬钩子单列出来。
- 这样会导致后续任何人都能写出一个“玩法上说得通，但原著味道不够”的升仙系统。

缺的不是数值，而是对象边界：

- 三气语义
- 升仙时的灾劫语义
- 福地 / 洞天的层级与质量语义

建议处理：

- 后续回改时，不一定现在就写公式和数值，但至少要把这三类状态写进长期对象清单。
- 否则架构一旦成型，升仙部分很容易只能靠后补字段。

#### 4. 仙窍经营规格没有把“时间流速 / 灾劫周期 / 福地-洞天差异”列为主循环核心

定位：

- `docs/superpowers/specs/2026-04-26-reborng-immortal-aperture-management-spec.md:52-70`
- `docs/superpowers/specs/2026-04-26-reborng-immortal-aperture-management-spec.md:95-103`
- `docs/superpowers/specs/2026-04-26-reborng-immortal-aperture-management-spec.md:141-174`

判断：

- 现文档已经正确强调了“仙窍不是背包，也不是普通种田系统”。
- 但它对仙窍的主循环定义还偏抽象经营层，缺少原著里最决定节奏和压力的几根硬骨架。

关键缺漏：

- 仙窍时间流速或等效时间压力
- 灾劫 / 经营灾变的正式周期位
- 福地与洞天等更高层结构差异的保留口

风险：

- 后续如果只按“资源产出 + 维护消耗 + 过载风险”做，玩法上能跑，但会越来越像泛化经营模拟，而不是《蛊真人》里的仙窍体系。

建议处理：

- 后续回改时，把“时间性压力”和“周期性大风险”写成仙窍层的一等状态，而不是放在文本描述里默认存在。

#### 5. 架构输入清单还没有明确保留“道痕 / 流派 / 灾劫 / 喂养负担 / 宝黄天接入媒介”对象

定位：

- `docs/superpowers/specs/2026-04-26-reborng-architecture-input-checklist.md:37-53`
- `docs/superpowers/specs/2026-04-26-reborng-architecture-input-checklist.md:82-129`

判断：

- 当前架构输入清单已经把大对象分层做得不错。
- 但从原著一致性的角度看，仍少了几类一旦晚加就会很痛的对象。

建议至少补出以下保留位：

- 主修流派 / 道痕演进对象
- 蛊虫喂养与维护负担对象
- 灾劫 / 周期性大风险对象
- 宝黄天接入与交易媒介对象

原因：

- 这些对象现在没有被明文保留，后面最容易被顺手塞进 `Build`、`资源` 或 `时间` 的已有字段里，最后又回到“一个字段承受多阶段语义”的老问题。

### P3 建议补强

#### 6. `十转` 与 `尊者级 IF` 的措辞现在可接受，但要持续锁死在高自由 IF 语境里

定位：

- `docs/superpowers/specs/2026-04-26-reborng-a-scheme-design.md:22`
- `docs/superpowers/specs/2026-04-26-reborng-a-scheme-design.md:80-84`
- `docs/superpowers/specs/2026-04-26-reborng-longterm-extension-state-boundary-spec.md:79-82`

判断：

- 目前文档已经写明这不是 `canon_strict` 的默认承诺目标，所以不构成当前硬冲突。
- 但这个措辞本身非常容易在后续文档传播时被误读成“项目默认终局必须做成十转爽游”。

建议处理：

- 继续保留这条时，后续所有涉及高阶终局的文档都应重复一句：仅属于高自由 IF 预留，不可反向决定前中期 canon 语义。

#### 7. 杀招、阵、仙蛊屋规格还缺一层更贴原著的组成约束提示

定位：

- `docs/superpowers/specs/2026-04-26-reborng-killer-move-formation-gu-house-spec.md:53-89`
- `docs/superpowers/specs/2026-04-26-reborng-killer-move-formation-gu-house-spec.md:151-180`

判断：

- 当前文档已经把三者从“内容资产”提升成了“系统层”，这一步非常关键，也做对了。
- 但从原著味道看，后续还应进一步补强它们的组成边界，例如：哪些依赖核心蛊，哪些依赖场域结构，哪些依赖长期供养与部件关系。

建议处理：

- 这不是现在就得补完的 `P2`，但如果相关系统即将进入架构正文，最好先补一版“更贴原著的组成约束页”。

## 没发现的问题

以下几类风险，本轮没有发现明显撞墙：

- 没有把黑市误写成宝黄天前身
- 没有把药堂债、人情债和蛊仙交易信用直接混写
- 没有把仙窍写成单纯背包或地图皮肤
- 没有把阵和仙蛊屋降格成普通装备栏
- 首发对方源等核心人物的 guarded 处理总体是稳的

## 对正式架构的影响判断

当前结论是：**仍可进架构，但要把本审阅中的 `P1 + P2` 当成架构前补钉清单。**

原因：

- 当前文档体系已经足够支撑对象分层、模块切割和存档演进设计
- 未发现会迫使首发底座整体推翻的硬冲突
- 发现的问题主要集中在“原著关键机制尚未被保留成一等状态”，而不是“现有方向完全错了”

不建议的做法：

- 直接把当前文本当作 canon-ready 定稿
- 在正式架构里继续沿用“路径倾向”混写
- 在交易和仙窍实现时，把缺的原著钩子临时塞进已有字段

建议的下一步：

1. 由你先审这份 closeout review，确认这些 finding 的定级是否符合你的预期。
2. 你确认后，再开一轮只改规格正文的“canon 补钉修订”。
3. 修完再进入正式架构文档，会更稳。
