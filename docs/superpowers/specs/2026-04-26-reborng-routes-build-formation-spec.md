## 文档角色

本文档负责定义首发到中后期的路线语义与 Build 形成逻辑，回答以下问题：

- 本作的 Build 到底由什么构成
- 为什么不能开局选职业，而要由机会与压力逼出路线
- 首发四主路线与药堂半主路线如何形成可感知分化
- Build 如何从凡人期自然扩展到杀招、阵、仙蛊屋与更高阶段

## 实现前公共接口问题

- 该系统的主循环是什么，最小操作单位是什么
- 它消耗哪些：时间、AP、资源、暴露、债务、人情、空间占用
- 它产出哪些：成长、材料、资格、部署能力、交易能力、关系变化
- 它依赖哪些上位状态：修为、空间、节点、路线、Build、资产所有权
- 它的失败形态是什么：受损、拖慢、欠账、暴露、路线断裂、资产损失
- 它在 `canon_strict` 与 `sandbox_if` 下哪里不同
- 它要在账本 UI 上最少显示哪些信息
- 它如何与首发系统兼容，而不是推翻首发系统

## 设计目标

### 1. 不做职业选择题

- 玩家不在开局点一个职业卡
- 路线和 Build 应由出身、抢到的蛊、伤势、债务、资源来源、门路和传承线共同逼出

### 2. 路线不是剧情标签

- 月光、功绩、黑市、传承不是叙事 flavor
- 它们必须对应不同风险结构、成长节奏和失败成本

### 3. Build 必须包含关系与债务

- Build 不能只写成“我带哪几只蛊”
- 谁保你、谁盯你、你靠什么活、你欠谁，都属于 Build

### 4. 未来扩展必须不重写定义

- 杀招、阵、仙蛊屋、仙窍经营都应视为 Build 的后续层
- 不能到了中后期又发明另一套“真正 Build”

## Build 总定义

本作的 Build 由以下八层共同组成：

- 求活路线层
- 主修流派 / 道痕层
- 核心蛊组合
- 支撑手段
- 关系与债务结构
- 资源来源结构
- 风险偏好与求活方式
- 维持与喂养负担

## 求活路线层

### 1. 求活路线不是道统完整定义

首发的求活路线不是完整修炼体系，而是“你更像怎么活”。

例如：

- 月光稳进
- 功绩积累
- 药堂粘合
- 黑市投机倾向
- 传承跳跃倾向

### 2. 首发求活路线的作用

- 影响你更适合去哪些节点
- 影响你更容易在哪类遭遇中活下来
- 影响你更常被什么资源咬住

## 主修流派 / 道痕层

### 1. 主修流派不等于首发求活路线

- 求活路线解决“你靠什么活”
- 主修流派解决“你往哪条长期修炼主轴走”

### 2. 首发不必完全展开，但必须保留

中后期至少要保留以下语义：

- 当前主修流派
- 兼修是否带来额外压力
- 道痕演进如何影响更高阶 Build

### 3. 为什么现在就要保留

- 升仙后，杀招、阵、仙蛊屋和高阶组合不再只看“你现在怎么活”
- 它们也会看你长期往哪类流派和道痕方向倾斜

## 核心蛊组合

### 1. 蛊组合是 Build 核心，但不是 Build 全部

首发 Build 至少要记录：

- 核心蛊
- 辅助蛊
- 暂时缺口
- 当前修行投入重点

### 2. 中后期兼容

到了更高阶段，Build 可自然叠加：

- 杀招模板
- 阵位需求
- 仙蛊屋部件位
- 仙窍产出支撑

这些是“Build 的后续层”，不是另起新系统。

## 支撑手段

支撑手段包括：

- 药理与恢复方式
- 侦查方式
- 关系借力方式
- 资源节流方式
- 脱身方式

它们决定同一套蛊组合为什么会活出不同感觉。

## 关系与债务结构

### 1. 关系是 Build 的组成部分

至少要记录：

- 谁能保你
- 谁会盯你
- 谁能介绍门路
- 谁要求你站队

### 2. 债务也是 Build 的组成部分

至少要记录：

- 你欠谁元石或药堂账
- 你欠谁人情
- 你欠谁未来动作

这部分在首发非常关键，中后期也不能丢。

## 资源来源结构

Build 还必须回答：

- 你主要靠什么养自己
- 你主要靠什么推动成长
- 你资源最怕被谁断

首发典型资源来源：

- 学堂修行
- 功绩换取
- 药堂置换
- 黑市偏门
- 传承残线

## 风险偏好与求活方式

同样重要的问题是：

- 你更像稳活、赌跳、靠人、靠伤换路，还是靠夜路翻身

这层决定：

- 你会如何进入遭遇
- 你会如何跑路
- 你会如何处理重创

## 首发路线定义

### 1. 月光路线

关键词：

- 制度内修行
- 学堂竞争
- 正面成长

优势：

- 方向明确
- 成长感强

代价：

- 受比较
- 受审视
- 容错不高

### 2. 功绩路线

关键词：

- 制度内资源争夺
- 稳健积累
- 可控但受审计

优势：

- 稳
- 易理解

代价：

- 慢
- 更容易留下秩序性痕迹

### 3. 黑市路线

关键词：

- 边路机会
- 高风险翻身
- 替代路径

优势：

- 机会大
- 能救急

代价：

- 暴露高
- 尾巴重
- 后续纠缠深

### 4. 传承路线

关键词：

- 制度外跳跃
- 高风险偏航
- 不稳定但可能改命

优势：

- 可能打破正常节奏

代价：

- 不真
- 易误导
- 最容易把人带进高危局

### 5. 药堂半主路线

关键词：

- 恢复
- 材料
- 欠账
- 人情交换

药堂不是独立大主线，而是最重要的咬合层之一。

## 首发 8 回合内的 Build 形成

### 回合 1-2

- 给出制度内入口
- 给出恢复压力
- 给出第一轮资源疼痛

### 回合 3-4

- 让玩家发现自己更自然会靠哪种资源活
- 让药堂、功绩、月光和黑市开始撕扯

### 回合 5-6

- 让遭遇、伤势和跑路方式显著影响 Build 倾向
- 让玩家开始放弃某些不适合自己的路

### 回合 7-8

- 让“我现在更像哪种活法”显性化
- 让核心蛊、债务、资源来源和风格形成阶段性组合

## Build 演进

### 1. 首发演进不求封死

首发 Build 要做的是“形成倾向”，不是“点满大树”。

### 2. 演进的三类来源

- 抢到或养出的核心蛊
- 被迫接受的关系与债务
- 因风险与收益反复验证后留下的活法

### 3. Build 的锁定与漂移

Build 应允许：

- 早期轻漂移
- 中期形成主轴
- 后期叠加更重的结构层

不应允许：

- 每回合无代价横跳成完全不同活法

## 中后期兼容接口

为了避免后续重构，Build 未来必须允许叠加以下层：

- `流派层`：主修流派、兼修压力、道痕演进
- `战斗层`：杀招模板与冲突风格
- `场域层`：阵与部署位
- `载体层`：仙蛊屋部件与激活方式
- `经营层`：仙窍产出、材料供给、养成负担
- `维持层`：蛊虫喂养、供养链、长期维护负担
- `交易层`：挂单、门路、组织支持

这些都属于同一 Build 的延展，而不是新角色页。

## 模式差异

### canon_strict

- 更少离谱捷径
- 路线分化更依赖原著证据与高可信推断
- 黑市、传承与高跳跃路线更受门槛控制

### sandbox_if

- 更允许激进偏航
- 更早接触某些危险 Build 组合
- 更允许关系与传闻把角色拽出制度常轨

## 账本 UI 最低要求

Build 页至少应让玩家看清：

- 当前主求活路线
- 当前主修流派或其保留位
- 核心蛊与支撑蛊
- 主要资源来源
- 当前债务与关键关系
- 当前更适合的节点或活法
- 当前 Build 的缺口和风险

## 验收标准

- 玩家在前 8 回合内能感到至少两条路线的真实结构差异
- 玩家能理解 Build 不只是蛊组合，还包括债务、关系和资源来源
- 玩家能在阶段结束时说出“自己更像哪种活法”
- 首发 Build 定义可自然挂接未来杀招、阵和仙蛊屋

## 失败信号

- 路线像职业标签
- Build 页只有蛊列表
- 玩家可以无代价同时吃满四条主路线
- 到中后期必须另起一套“真正 Build 系统”
