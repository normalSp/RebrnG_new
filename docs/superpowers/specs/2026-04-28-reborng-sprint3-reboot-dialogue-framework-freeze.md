## 文档角色

本文冻结 Sprint 3 的目标效果和架构边界。Sprint 3 的目标不是继续扩长局，也不是接入 runtime AI，而是把当前 8 自由窗口底座前置一层“人生重开设置”，并在呈现层改成“对话流主界面 + 账本明细侧栏”。

`example.md` 的可取之处是体验结构：开局天赋 / 属性生成、长正文剧情推进、强叙事选项、选择后大段结果反馈、阶段式人生推进。它的问题是大量超模天赋、无代价抢原著机缘、提前改写方源硬事实、成功率和概率大幅直出。Sprint 3 只学习形式，不照搬这些会破坏 `canon_strict` 的内容。

## 核心结论

- 项目有能力做成接近豆包式人生重开体验，且不需要推翻现有 Rust 规则底座。
- UI 最终目标形态调整为“对话流主界面 + 账本明细侧栏”，不是纯聊天，也不是纯策略表格。
- 所有选择仍必须进入 `ActionCommand` 管线，规则事实仍由 Rust 写入。
- DeepSeek / AI 不进入玩家点击后的同步 runtime 主链；后续只能作为离线内容扩写工具。
- `canon_strict` 维持压迫、锚点保护和原著严谨；高自由强改写只允许进入 `sandbox_if`。

## 目标体验

### 主界面形态

主界面以正文对话流为第一阅读层：
- 阶段标题：例如“开窍大典余波”“学堂立足”“黑市暗口”。
- 当前正文：解释局势、行动结果、NPC反应和下一步压力。
- 可选行动：使用强叙事短标题和一段风险说明。
- 最近结果：显示上一选择导致的明确变化。

账本侧栏为第二阅读层：
- AP、时段、窗口、地点。
- 元石、材料、功绩、债务、人情。
- 暴露、伤势、遭遇、黑市门路。
- 空窍状态、修行痕迹、蛊虫状态、Build 层。
- 线索、传闻、证据等级、模式许可。

### 行动反馈

一次行动完成后，必须同时产生三类反馈：
- 对话流正文：告诉玩家发生了什么。
- 结果摘要：告诉玩家这次点击已经成功落账。
- 账本明细：告诉玩家哪些可定量状态发生变化。

玩家不应该需要翻因果账本才能知道自己是否移动成功、是否修行成功、是否承担了债务或暴露。

## 人生重开设置层

Sprint 3 新增设置层，但不替代 S0 底座。

设置层最小对象：
- `RunSetupState`：保存当前是否处于设置阶段、候选项、已选项、模式、设置完成状态。
- `OriginSpec`：出身，决定初始社会关系、资源来源、安全区和压力来源。
- `TalentSpec`：天赋，决定属性修正、路线倾向、线索可见性或 IF 许可。
- `AttributeProfile`：初始属性面板，至少覆盖资质、气运、心智、根骨。
- `OpeningRiteOutcome`：开窍大典结果，决定空窍已开、资质展示、资源包和关注度。
- `InitialResourcePackage`：元石、基础物资、初始债务或人情。

设置完成后进入现有 S0 青茅山 8 自由窗口循环。当前固定开局改为“默认预设开局”，继续用于测试、调试和快速走查。

## 模式门禁

### canon_strict

`canon_strict` 下允许：
- 低风险出身差异。
- 资质、根骨、心智、气运的温和修正。
- 开窍大典中的展示策略、资源包差异、关注度变化。
- 不改写原著硬事实的线索可见性变化。

`canon_strict` 下禁止：
- 开局获得完整花酒行者传承。
- 开局稳定获得本命蛊或高阶蛊虫。
- 直接提前击杀、绑定、结盟或硬改写方源等核心人物。
- 用超模天赋抹掉生存压迫、债务、暴露、伤势和锚点压力。
- 把成功率公式和概率堆成爽文数值展示。

### sandbox_if

`sandbox_if` 下允许更高自由度：
- 强天赋、重生者、提前识破部分机缘。
- 更高比例原创路线和原著锚点偏移。
- 更大胆的 IF 选择和阶段性强奖励。

但仍必须受以下门禁限制：
- 阶段门槛。
- 资源和风险代价。
- 主线保护。
- 世界空间和行动窗口。
- 蛊虫、杀招、传承和本命蛊的状态合同。

## 定量表达

Sprint 3 采用“账本明细”方案：
- 明确显示状态变化，例如“元石 -1”“暴露 +2”“药堂债 +1”“月光修行痕迹 +1”。
- 不默认公开所有成功率、概率、公式和隐藏变量。
- 关键失败或危险行动必须给出定性风险，例如“可能压缩后续窗口”“可能引来公开压力”“会抬高黑市暴露”。
- 隐藏世界变量仍只通过征兆、风声、人物反应和因果账提示呈现。

## 对话流投影

新增 `DialogueTimelineView` 或等价投影，作为 UI 读取的对话流模型。

最小字段语义：
- 当前阶段标题。
- 当前正文段落。
- 上一次选择标题。
- 上一次结果摘要。
- 可选行动列表。
- 最新账本变化摘要。
- 当前模式和证据门禁提示。

`DialogueTimelineView` 只允许从规则状态、内容 bundle 和因果账本派生。React 不得自行生成剧情，不得自行判断奖励，不得直接改写状态。

## 开窍大典锚点

Sprint 3 将开窍大典提升为设置层与 S0 循环之间的强制开场锚点。

冻结规则：
- 设置层可以决定资质展示、初始资源、关注度和关系压力。
- S0 自由窗口仍从“空窍已开、根基未稳”之后开始。
- 若未来实现开窍前短序章，序章中修行行动必须不可用。
- 空窍状态不得和本命蛊、核心蛊、辅助蛊混写。

## DeepSeek 与内容扩写边界

DeepSeek 后续只允许进入离线内容扩写管线：
- 生成候选正文、行动说明、结果反馈和风声文本。
- 由人工或内容工具筛选、打标签、加证据等级、写入 YAML。
- 经内容构建器校验后进入 bundle。

DeepSeek 禁止：
- 玩家点击后同步生成规则结果。
- 运行时生成奖励、蛊虫、传承、锚点变化或原著硬事实。
- 直接读取或写入 `GameState`。
- 将 API key、prompt、response 写入存档或仓库。

## Sprint 3 推荐阶段

1. Phase 1：冻结本文档，作为 Sprint 3 目标效果基线。
2. Phase 2：新增设置层内容合同，定义出身、天赋、属性、开窍大典结果。
3. Phase 3：实现最小 `RunSetupState` 与默认预设开局兼容。
4. Phase 4：实现 `DialogueTimelineView` 与对话流 UI 壳。
5. Phase 5：把现有 S0 8 窗口用对话流方式完整跑通。
6. Phase 6：冻结 DeepSeek 离线扩写管线方案，但不接 runtime 主链。

## 验收标准

- 新开局能先进入人生重开设置层，再进入 S0 8 自由窗口。
- 默认预设开局仍能绕过设置层，用于自动测试和快速走查。
- `canon_strict` 拒绝高强度 IF 天赋和硬改写原著机缘。
- `sandbox_if` 可以启用强天赋，但仍受阶段、资源、风险和锚点门禁限制。
- 同一次行动在 UI 上同时产生正文结果、点击成功反馈和账本明细变化。
- 进入 S0 后，AP、移动、债务、暴露、伤势、黑市解锁、遭遇、存档仍通过现有规则管线。
- 运行时无 AI proposal / narrator 参与 `resolve_action`、Tauri command 或 UI 点击主链。

## 失败信号

- 为了像豆包而把规则结算交给 AI。
- 为了爽感在 `canon_strict` 中开放超模天赋、完整传承或原著硬改写。
- 设置层生成的状态不能进入现有存档和 ActionCommand 管线。
- UI 只显示长正文，但账本明细缺失，导致可定量机制退化。
- 对话流直接替代账本母体，导致中后期仙窍、宝黄天、阵、仙蛊屋无法接入。

## Assumptions

- `example.md` 是体验参考，不是内容源，不在本轮提交中纳入仓库。
- Sprint 3 先做目标冻结和框架，不直接扩写大量剧情。
- 当前 8 窗口首发底座继续保留。
- 强天赋、重生者、提前抢机缘等只进入 `sandbox_if`。
