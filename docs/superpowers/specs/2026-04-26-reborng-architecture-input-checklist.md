## 文档角色

本文档不是正式架构设计，而是把现有规格转译成“架构前必须看见的输入清单”，避免后续正式设计时漏掉核心对象和边界。

## 规格覆盖摘要

当前已具备的规格组：

- 总纲
- 首发纵切
- 核心循环 / 时间 / AP
- 节点地图 / 移动 / 暴露 / 安全区
- 经济 / 债务 / 药堂 / 功绩 / 黑市
- 遭遇战 / 跑路 / 重创可续
- 路线与 Build 形成
- 账本式 UI 信息架构
- 长期扩展约束与状态边界
- 修为阶段推进与升仙
- 仙窍经营
- 跨图 / 地区切换 / 世界空间
- 蛊仙交易 / 宝黄天
- 杀招 / 阵 / 仙蛊屋
- 势力依附 / 组织委托 / 中后期关系债

## 架构必须显式建模的核心状态对象

- 修为阶段状态
- 时间窗口状态
- 节点与路线状态
- 暴露状态
- 伤势与恢复状态
- 经济资源状态
- 债务与信誉状态
- 关系与势力状态
- Build 状态
- 资产容器状态
- 世界空间状态
- 仙窍经营状态
- 交易挂单状态
- 杀招 / 阵 / 仙蛊屋状态
- 存档演进状态

## 模块边界最低要求

正式架构至少要能分清以下边界：

- 主循环层
- 世界空间层
- 资源与债务层
- Build 与成长层
- 冲突与后果层
- 经营与交易层
- 关系与组织层
- UI 账本呈现层
- 存档与演进层

## 子循环关系图要求

正式架构文档至少要说明以下子循环如何嵌套：

- 首发求活主循环
- 遭遇与跑路子循环
- 恢复与债务子循环
- 修为推进与升仙子循环
- 仙窍经营子循环
- 交易与挂单子循环
- 阵与仙蛊屋部署子循环
- 组织委托与关系债子循环

## 资产与容器要求

正式架构不能跳过以下容器语义：

- 随身
- 落脚点
- 仙窍内
- 组织托管
- 挂单中
- 部署中

## 资源分层要求

正式架构不能把以下层混成一层：

- 凡人层资源
- 蛊仙层资源
- 部署与组织层资源

## 空间分层要求

正式架构不能把以下空间混成一层：

- 局部节点图
- 地区层
- 跨图区
- 仙窍内空间
- 交易空间

## Build 分层要求

正式架构不能把以下 Build 层压扁成一页蛊列表：

- 路线与求活层
- 蛊组合层
- 杀招层
- 场域部署层
- 经营支撑层

## 存档演进要求

正式架构必须明确：

- 哪些状态短期刷新
- 哪些状态跨章节保留
- 哪些状态跨阶段保留
- 哪些状态在升仙时发生结构性迁移

## 架构前仍需执行的总收口

在正式架构文档开写前，建议做一次只读交叉审查，检查：

- 是否还有任何文档把黑市写成宝黄天前身
- 是否还有任何文档把债务和信用混写
- 是否还有任何文档把仙窍写成大背包
- 是否还有任何文档把阵和仙蛊屋写成普通技能或装备
- 是否还有任何文档需要一个字段承担多个阶段语义
