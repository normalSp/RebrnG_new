# RebrnG 第一卷剧情圣经与 IF 走向总纲

Status: frozen candidate  
Date: 2026-04-28  
Scope: Sprint 7.1

## Summary

本文件用于替代“哪里文本短就补哪里”的散点式推进方式，先把《蛊真人》第一卷《魔性不改》做成游戏可生产的剧情圣经。后续 DeepSeek V4-Pro 只能按这里定义的剧情段、锚点、玩家空间和红线扩写候选文本，不能自由发明主线事实。

只读核对 `reverend-insanity.txt` 后，第一卷范围确认为：从第一节“纵身亡魔心仍不悔”到第一百九十九节“走向各自的命运”，第二卷从“魔子出山”开始。第一卷核心体验不是青茅山日常，而是：

重生归来 -> 开窍 -> 月光蛊 / 酒虫 / 花酒遗藏 -> 学堂与家族压迫 -> 商队与贾金生 -> 传承推进 -> 兽潮与白凝冰 -> 青茅山终局与脱离。

当前 S0 8 自由窗口只对应第一卷前 10-15% 的可玩切片：开窍大典后、学堂入账、月光蛊根基、第一轮制度压力。它不是完整第一卷，也不应该承担商队、兽潮、白凝冰和终局内容。

## Canon Index Layer

Sprint 7.2 起，剧情圣经不再独自承载所有原著细节。第一卷细节证据拆到原著事实索引层：

- `docs/superpowers/data/canon/first-volume-source-map.json`：只记录本地原著 source 的章节行号、编码和 hash，不保存原文。
- `docs/superpowers/data/canon/first-volume-canon-index.json`：保存转述事实卡，例如开窍大典地下溶洞、水中行走、希望蛊入体、元海容量、真元颜色、炼化阻力和空窍壁膜推进。
- `docs/superpowers/data/first-volume-story-beats.json`：每个剧情 beat 通过 `canon_index_refs` 引用事实卡。

剧情圣经继续管主线、锚点、玩家空间和 IF 边界；原著事实索引管细节证据。DeepSeek V4-Pro 后续扩写时只能吃 `NarrativeSlot + CanonFactCard + 红线`，不得自由续写整本原文，也不得把长段原文写入仓库。

首批索引重点覆盖 Beat 1-2 与机制关键事实。后续每推进一个剧情段，再补对应事实卡，而不是一次性把第一卷 199 节全部细索引完。

## Product Direction

专业游戏开发判断：下一步应先冻结第一卷剧情大纲，再扩正文和机制。

原因：

- 没有剧情圣经时，DeepSeek 扩写会变成散文补丁，难以保证原著锚点、IF 代价和系统状态一致。
- 没有剧情段边界时，机制开发容易“有系统没剧情牵引”，比如先做炼蛊升转却不知道它服务哪段原著压力。
- 玩家想要的是“人生重开式长正文推进 + 可定性可定量规则”，所以每个剧情段都必须同时回答：读到什么、选什么、付什么代价、账本改什么。

推荐生产节奏：

1. 用本文件和 `first-volume-story-beats.json` 锁第一卷母线。
2. 每个 Sprint 只挑 1-2 个剧情段做可玩扩展。
3. DeepSeek 只生成 `NarrativeSlot` 候选，人工审校后入库。
4. 机制开发跟随剧情缺口，而不是独立横向铺系统。

## Core Rule

玩家不是方源，也不替代方源。玩家是同一青茅山秩序里的另一名新人蛊师，在原著锚点附近求活、借势、避险、误判、付代价偏移。

`canon_strict`：

- 保护原著硬事实。
- 玩家主要承受余波、经营缝隙、争取生存资格。
- 不允许早期抢走方源核心机缘、控制方源、改写方正资质对照、击败白凝冰或改写青茅山终局。

`sandbox_if`：

- 允许有限偏移。
- 每条偏移必须有代价、失败形态和锚点压力。
- IF 不是爽文超模路线，而是“更早接触危险、更深卷入因果、更难脱身”。

## Beat Design Details

### 1. 重生前夜与开窍大典

原著功能：建立方源回归、青茅山牢笼感、家族对资质的期待、方正对照、希望蛊开窍。

玩家体验：玩家以同届少年身份进入开窍大典，得到出身、资质、初始资源和关注度。开窍不是抽卡爽点，而是“从此被纳入制度账册”。

`canon_strict` 禁止：

- 玩家替代方源或方正开窍结果。
- 玩家未开窍修行。
- 开局稳定获得本命蛊、酒虫、完整传承或尊者级线索。

`sandbox_if` 可做：

- 异常梦兆。
- 希望蛊反应偏强。
- 资质或天赋带来额外关注。

机制缺口：空窍容量、资质等级、真元海、开窍仪式锚点。

### 2. 学堂入账与蛊师常识

原著功能：讲九转体系、蛊师常识、花酒行者传闻、月光蛊作为家族制度资源。

玩家体验：学堂既是教学，也是审计和分类。玩家领取月光蛊，知道蛊虫要炼化、要养、要承担损伤。

`canon_strict` 禁止：

- 把月光蛊写成普通装备。
- 直接获得花酒行者真相。
- 把学堂写成传统导师制。

`sandbox_if` 可做：

- 旁支听到花酒传闻异词。
- 月光蛊对玩家天赋有轻微异动。

机制缺口：蛊师常识图鉴、蛊虫喂养周期、学堂课程锚点。

### 3. 月光蛊与酒虫诱惑

原著功能：方源寻找酒虫，月光蛊作为保底选择，本命蛊概念进入叙事。

玩家体验：月光蛊是稳定制度路线；酒虫是危险诱惑，只能以传闻、风声、假线或 IF 残影出现。

`canon_strict` 禁止：

- 稳定获得酒虫。
- 直接成为酒虫发现者。
- 抢走方源对酒虫和花酒遗藏的核心路径。

`sandbox_if` 可做：

- 追到酒香残线但失败。
- 发现假入口。
- 得到“酒虫曾出现”的旁证，换来高层或方源余波风险。

机制缺口：酒虫 `GuSpec`、特殊炼化难度、本命蛊绑定风险、真假线索。

### 4. 花酒遗藏与历史真相

原著功能：方源发现花酒尸骸、获得酒虫、看见留影存声蛊揭示历史被胜利者篡改。

玩家体验：玩家不能拿主奖励，但可以接触历史裂缝：听到旧说法矛盾、看到外围痕迹、误入假线索，形成“原著味”的危险好奇。

`canon_strict` 禁止：

- 完整获得花酒遗藏。
- 拿走酒虫、留影存声蛊或核心元石。
- 公开改写家族历史真相。

`sandbox_if` 可做：

- 获得破碎证据。
- 被高层盯上。
- 与方源路线擦肩，引发锚点压力。

机制缺口：传承门禁、证据等级、历史真相揭露风险、高层清洗追索。

### 5. 早期学堂竞争

原著功能：月刃训练、近战、组织本质、勒索、班头、同辈压迫和制度利用。

玩家体验：玩家不需要复制方源勒索路线，但要感到同样的压迫逻辑：资源短缺、名声压力、被审计、同辈试探、老师利用。

`canon_strict` 禁止：

- 爽文打脸式横扫学堂。
- 玩家取代方源班头线。
- 学堂压力无代价解除。

`sandbox_if` 可做：

- 局部反制一名同辈。
- 嫁祸或拖延。
- 通过药堂、黑市、功绩线规避部分正面冲突。

机制缺口：名声、同辈关系、组织审计、班级压力、非致命对练。

### 6. 商队与贾金生事件

原著功能：商队带来外部资源，赌石与蟾蛊引出贾金生死亡，家族与商队审查升级。

玩家体验：商队是机会也是审查。玩家可以接触赌石、交易、外来消息、调查压力，但不能主导贾金生核心案件。

`canon_strict` 禁止：

- 改写贾金生死亡核心事实。
- 稳定夺取原著赌石收益。
- 让玩家控制商队调查结论。

`sandbox_if` 可做：

- 被误认为目击者。
- 得到边缘证物。
- 因黑市或交易记录被卷入审查。

机制缺口：外来商队交易、调查状态、证词/嫌疑、赌石高风险系统。

### 7. 花酒传承推进

原著功能：壁破甬道、蛊室再选蛊、玉皮蛊、白豕蛊、红圈标记、战力大涨。

玩家体验：传承线从“传闻诱惑”升级为“外围残线”。玩家可以追残线、错线、假线或受到传承余波影响，但主奖励仍需锚点保护。

`canon_strict` 禁止：

- 稳定获得白豕蛊、玉皮蛊等主线奖励。
- 直接进入花酒核心密室。
- 抹掉方源推进传承的硬事实。

`sandbox_if` 可做：

- 获得残缺辅助材料。
- 被假红圈误导。
- 以重伤、债务或高暴露换取边缘收益。

机制缺口：传承层级、残线真假、蛊室门禁、奖励替代池。

### 8. 年末考核与家族打压

原著功能：年末考核，方源战力显露，家族怀疑、试探、刁难和打压逐步加重。

玩家体验：阶段性考核开始检验玩家路线。玩家可以争排名、藏拙、借债、养蛊或走偏门，但每种选择都会写入家族视线。

`canon_strict` 禁止：

- 取代方源关键表现。
- 轻易化解家族怀疑。
- 无代价获得高名声。

`sandbox_if` 可做：

- 改变玩家自身考核结果。
- 与方正、漠北、赤城等同届关系局部偏移。

机制缺口：考核系统、名声/怀疑、派系压力、阶段评价。

### 9. 小兽潮与山野成长

原著功能：小兽潮、山野杀戮、白豕力量、猴群、炼蛊艰难、二转推进。

玩家体验：地图从山寨内扩到山野。移动不再只是换节点，而是风险、补给、伤势、蛊虫损伤和撤退判断。

`canon_strict` 禁止：

- 把兽潮写成刷怪收益。
- 稳定提前解除狼潮/兽潮大势。
- 跳过山野死亡压力。

`sandbox_if` 可做：

- 提前得到征兆。
- 低概率救下一名边缘 NPC。
- 高代价绕开一次危险但留下后患。

机制缺口：山野风险表、野兽遭遇、长跳时、战败资产损失、蛊虫损伤。

### 10. 白凝冰与三寨压力

原著功能：白凝冰登场，北冥冰魄体，青书牺牲，三寨压力和强者尺度明确。

玩家体验：玩家第一次明确感到“世界不是围着自己转”。白凝冰是压力源，不是可攻略 NPC 或早期 Boss。

`canon_strict` 禁止：

- 稳定击败、收服、控制白凝冰。
- 让玩家早期与白凝冰同层竞争。
- 改写青书牺牲等核心压力结构。

`sandbox_if` 可做：

- 短暂遭遇白凝冰余波。
- 被强者利用或误伤。
- 以重伤/断线换取一条信息。

机制缺口：guarded NPC、高阶压力、势力事件、不可战胜遭遇逃生。

### 11. 青茅山终局

原著功能：狼潮、血湖墓地、血海老祖、一代古月、鹤灾、方源脱离、再次重生前后的终局推动。

玩家体验：阶段终点不是赢下青茅山，而是带着资产、蛊虫、伤势、债务、关系断裂或 IF 偏移痕迹活下来。脱离资格是第一卷的阶段成功。

`canon_strict` 禁止：

- 无代价改写青茅山大结局。
- 阻止核心灾变。
- 抢走一代古月/方源/白凝冰等核心命运主导权。

`sandbox_if` 可做：

- 局部人物余波改变。
- 玩家资产容器迁移不同。
- 以长期追索换取偏移幸存路线。

机制缺口：阶段结算、跨地图迁移、长期状态继承、资产容器转移、锚点压力清算。

## DeepSeek Expansion Rules

DeepSeek V4-Pro 后续扩写必须遵守：

- 输入必须是 `NarrativeSlot`，不是整本自由续写。
- 每个候选必须标注 `mode`、`evidence`、`state_assumptions`、`redlines`。
- 候选只进 `.local/deepseek-candidates/`，人工审校后才允许入库 YAML。
- 不保存 key、prompt、完整 response、模型名或 thinking chain。
- 不让 AI 生成规则事实；资源、蛊虫、伤势、债务、锚点状态仍由 Rust 规则层决定。
