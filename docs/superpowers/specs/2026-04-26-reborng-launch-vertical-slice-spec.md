# RebrnG 首发纵切规格 v1.0

状态：frozen candidate

更新时间：2026-04-26

关联母文档：[2026-04-26-reborng-a-scheme-design.md](./2026-04-26-reborng-a-scheme-design.md)

## 文档角色

本文件不是总设计重写，而是把 A 方案压成第一份可开发、可验证、可拆任务的首发纵切规格。

本文件回答五个问题：

- 第一份可玩版本到底验证什么
- 第一份可玩版本明确做什么、不做什么
- 玩家在 60-120 分钟里会经历什么
- 哪些系统必须在纵切里形成闭环
- 后续团队用什么标准判断纵切是否成立

## 纵切定位

### 1. 纵切性质

- 这是首发方向下的第一份内部可玩纵切，不是公开演示版，也不是最终首发全部内容。
- 它必须能代表正式产品的基因，而不是做一个以后会被废弃的原型玩具。
- 它优先验证“核心求活循环”，不是优先验证“最大自由度”。

### 2. 纵切目标时长

- 单次完整体验目标时长：60-120 分钟。
- 同一玩家允许在阶段存档下分两次体验完。
- 纵切必须能在这段时长内让路线分化、经济压力、黑市诱惑、药堂债、跑路价值和锚点保护都被感知到。

### 3. 纵切起始形式

经确认，本纵切采用：

- 单固定开局

设计含义：

- 先用一个固定身份、固定起始局面，把前 8 回合的核心循环、路线分化、债务压力、跑路与重创可续验证扎实。
- 不在首个纵切里验证完整自由开局。
- 不在首个纵切里验证 2-3 套完全不同的起始身份。

## 纵切成功定义

只要同时满足以下四条，就认为首发纵切成立：

- 玩家能在前 8 回合内明显感到“家族秩序下的求活压力”，而不是普通新手村成长。
- 玩家能做出至少两次真正困难的资源取舍，而不是线性推进。
- 玩家能清楚感到四条主路线并非装饰标签，而是不同风险结构。
- 玩家在一局结束时，不论成功或失败，都能留下清晰的“这不是轻量人生重开，而是冷峻因果生存 RPG”的印象。

## 玩家承诺

纵切向玩家承诺的不是“大量内容”，而是以下体验：

- 你不是世界中心，只是青茅山秩序下一个很容易被榨干的年轻修行者。
- 你每一步都在制度缝隙里找活路，而不是随便点点就能强起来。
- 你可以走正路，也可以摸边路，但所有路都要付代价。
- 你很早就能感到黑市、债务、传承、月光修行和功绩体系在互相拉扯。
- 你经常会发现“活下来”比“赢一场”更重要。

## 固定开局定义

### 1. 主角身份

纵切固定主角为：

- 古月一族旁支少年
- 年龄固定为 15
- 非原著核心人物
- 允许玩家改名，但不允许更改身份框架

### 2. 身份条件

该固定开局必须同时具备以下约束：

- 家族边缘身份，没有强后台
- 已被纳入学堂体系，无法轻易脱离制度
- 开局就背有轻度药堂债或家庭性人情债
- 资源偏紧，没有舒适的开局元石储备
- 与方源、方正、漠北等原著核心同代人处于危险邻近，但不深度绑定

### 3. 资质与修行状态

纵切默认：

- 开局通过短锚点进入开窍前夜
- 很快进入开窍大典
- 开窍后进入一转前后的早期根基阶段
- 默认采用中下压力资质，推荐为丙等

采用丙等的原因：

- 足够有压力
- 不会把纵切做成纯受苦片
- 可以让月光、功绩、药堂、黑市、传承都保有真实选择价值

### 4. 起始资源建议

纵切不要求在本文件里钉死最终数值，但要求起始状态满足以下体感：

- 元石非常紧，不足以无脑恢复和试错
- 药堂债能在前 3-5 回合内产生真实影响
- 家中落脚点很简陋，只能承担藏一点东西和喘口气
- 开窍后拿到制度内标准成长入口，但远远不够安全

## 纵切体验结构

### 1. 体验总结构

纵切由三段组成：

- A 段：短锚点导入
- B 段：受控进入主循环
- C 段：8 回合求活闭环

### 2. A 段：短锚点导入

目标：

- 建立世界压迫感
- 明确你不是主角团
- 明确开窍是制度入口，不是祝福

必须包含：

- 开窍前夜的家族/家庭/旁支压迫感
- 方源重生余波只作为风声或视线边角出现
- 开窍大典作为强制锚点推进

要求：

- 时长控制在 10-15 分钟
- 不允许玩家在这一段获得虚假的“大自由”
- 这一段主要负责立调性，不负责让玩家以为自己在看长篇剧情小说

### 3. B 段：受控进入主循环

目标：

- 让玩家理解账本、节点、AP、风险暴露、债务和修行状态如何一起工作
- 让玩家拿到第一个真正有代价的选择

必须出现：

- 学堂与家族制度内的第一轮资源/地位压力
- 月光路线入口
- 药堂路线入口
- 功绩路线入口

要求：

- 这段应该比 A 段更可操作
- 必须让玩家在第一个自由回合前，已经知道“没有哪条路是纯白送的”

### 4. C 段：8 回合求活闭环

这是纵切核心。

8 回合内必须完成：

- 资源紧张显性化
- 债务追索显性化
- 四主路线至少出现入口级分化
- 至少一次危险遭遇
- 至少一次“跑路比硬打更优”的真实局面
- 至少一次传承残线或黑市残线诱惑
- 至少一次锚点保护感知
- 至少一种阶段性结果收束

## 8 回合压力曲线

### 回合 1-2

目标：

- 建立学堂/家族秩序压力
- 建立元石与恢复的紧张感
- 让玩家先看到正路入口，再闻到边路气味

玩家应感知到：

- 月光路线是最显眼的制度内成长口
- 功绩路线是更稳但更受审计的资源路
- 药堂既是恢复口，也是负担口
- 黑市与传承还不是按钮，而是风声和可能性

### 回合 3-4

目标：

- 让路线差异开始变得实质化
- 让药堂债或家庭债开始真实压迫决策

玩家应感知到：

- 继续走学堂正路会被拿去和别人比较
- 试图省资源会影响恢复与修行节奏
- 黑市门路不是随便点亮，而是会带来暴露风险
- 传承线不是福利，而是高风险偏航

### 回合 5-6

目标：

- 推出第一轮真正危险局面
- 验证跑路、拖延、换场、求饶等生存技能

玩家应感知到：

- 一次错误准备会让自己掉进重创可续，而不是简单 Game Over
- 一次谨慎跑路可能比赢一场更值
- 药堂、人情或功绩可以救命，但都会留下账

### 回合 7-8

目标：

- 让 Build 倾向显性化
- 形成局部结算

玩家应感知到：

- 自己更像月光/功绩/黑市/传承哪条路
- 债务、名声、受伤、暴露和资源彼此联动
- 站稳一转根基不是爽点，而是从“随时可能被碾”进入“勉强能自立”

## 纵切包含的系统

### 1. 节点地图系统

必须包含：

- 小范围节点地图
- 节点间移动成本
- 不同时段的风险差异
- 少量勉强安全点
- 部分隐藏节点或隐藏入口

首发纵切推荐节点数：

- 7-9 个

推荐节点池：

- 学堂前院
- 学堂修行点
- 功绩登记处
- 药堂外厅
- 旁支住处/落脚点
- 山寨巷道
- 山野边缘/青茅竹线
- 酒肆或风声点
- 隐藏黑市接入口

### 2. 时间与 AP 系统

必须包含：

- 每个自由回合的可见 AP
- 场景行动对 AP 的消耗
- 关键章节窗口对自由行动的强制打断或覆盖

首发纵切建议：

- 每自由回合 2 AP
- 少量重动作可消耗 2 AP
- 关键锚点段可为 0 AP 强制触发

### 3. 经济与债务系统

必须包含：

- 元石压力
- 药堂债/人情债
- 基础恢复与调息成本
- 至少一种制度内资源获取方式
- 至少一种制度边缘资源获取方式

纵切要求：

- 不能让玩家觉得“钱只是数字”
- 必须让欠债、治疗、恢复、买消息、修行投入形成真实竞争

### 4. 路线形成系统

必须包含四条主路线的入口级验证：

- 月光
- 功绩
- 黑市
- 传承

以及一条半主路线：

- 药堂

要求：

- 纵切里不必把每条路线做成完整大树
- 但必须做出“这条路是怎么活、怎么赚、怎么被反噬”的差异

### 5. 遭遇战与跑路系统

必须包含：

- 至少 3 类短促危险遭遇
- 至少 1 类可通过侦查获得部分真实情报的遭遇
- 至少 1 类明显适合跑路的遭遇
- 重创可续状态

推荐遭遇模板：

- 学堂公开对练/羞辱压力
- 巷道堵人/勒索/试探
- 山野边缘误判风险/抢夺/伏击

### 6. 修行与空窍状态系统

必须包含：

- 开窍后的基础修行状态
- 月光路线相关的早期修行投入
- 调息恢复
- 轻度受损与修行节奏关系

要求：

- 首发纵切只需一转前后最小闭环
- 不引入高阶空窍经营
- 但必须让修行页真实参与决策，而不是装饰 UI

### 7. 风声、线索与传承残线系统

必须包含：

- 可记录的风声
- 至少一条半真半假的传承残线
- 至少一条会误导玩家但不至于纯坑杀的传闻
- 至少一个高风险高回报的外部诱惑

### 8. 锚点保护系统

必须包含：

- 开局短锚点
- 与原著核心人物有关的 guarded 信息处理
- 禁止直接改写的硬事实
- 玩家只能承接余波，不能首局就抢主线位置

### 9. 模式门禁系统

纵切必须同时验证两模式存在，但差异范围要受控。

`canon_strict` 必须验证：

- 缺证据交易不能直接落地
- 高风险补完内容不能假装原著既定事实
- 黑市通用交易不能伪装成完全开放商店

`sandbox_if` 必须验证：

- 可以比严格模式多开放少量边路机会
- 允许更强的黑市/传承诱惑
- 仍不得让主角轻易越过原著主线保护线

## 纵切不包含的系统

以下内容明确不进入首个纵切：

- 完整自由开局构筑
- 多起始身份并行验证
- 多区域移动
- 完整黑市商店与通用价格盘
- 完整药堂治疗清单
- 自由喂蛊大系统
- 高阶传承大副本
- 阵系统正式部署玩法
- 仙蛊屋组装与使用
- 升仙流程
- 仙窍经营
- 宝黄天交易盘
- 队友养成
- 恋爱养成
- 稳定基地经营

## 内容体量建议

### 1. 场景与节点

- 强制锚点场景：3-4 个
- 自由节点：7-9 个
- 隐藏入口或隐藏节点：1-2 个

### 2. NPC

- 核心可交互 NPC：8-12 个
- 其中 guarded 原著核心邻近 NPC：3-4 个
- 真正可形成深债/深压的关键 NPC：3-5 个

### 3. 遭遇与事件

- 危险遭遇模板：5-7 个
- 路线入口事件：每主路线至少 2 个
- 药堂半主路线事件：2-3 个
- 风声/传闻事件：6-10 个

### 4. 结果状态

纵切至少支持以下结果状态：

- 死亡
- 重创可续
- 债务压深但活着
- 名声受损但路线成形
- 勉强站稳一转根基

## UI 与信息架构最低要求

纵切必须验证账本式母界面是否成立。

最低需要可用的页面或侧页：

- 正文场景页
- 因果账本页
- 节点地图页
- 关系局势页
- 空窍/修行状态页
- 物资与债务页
- 线索与传闻页

要求：

- 页面可以是低保真
- 但信息组织不能是临时脚手架
- 玩家必须能感到这些页属于同一本账册，而不是七个彼此无关的菜单

## 纵切验收标准

### 1. 体验验收

- 试玩 20 分钟内，玩家能感到资源紧张与制度压力
- 试玩 40 分钟内，玩家能感到至少两条路线入口差异
- 试玩 60-120 分钟内，玩家能走到一轮阶段结果

### 2. 机制验收

- 6-8 回合内必须出现明显元石紧张、恢复紧张和债务压力
- 跑路在多数危险遭遇中必须是有效选择
- 战败后常见进入重创可续，而不是总是立即死亡
- 黑市在未解锁前不可见
- 黑市解锁后必须显著增加机会，但抬高暴露风险
- 药堂债必须能锁选项、改价格或改处置后果

### 3. 模式验收

- `canon_strict` 必须拒绝缺证据内容直接落地
- `sandbox_if` 必须多给出有限但真实的边路选择
- 双模式差异必须能感知，但不能像两款完全不同的游戏

### 4. 调性验收

- 文风不能变成现代吐槽
- 失败不能太轻飘
- 路线不能像传统职业树
- 原著核心人物不能被轻易降格成普通任务发放者

## Playtest 清单

每轮纵切测试至少回答以下问题：

- 玩家是否会在前 2 回合误以为这只是普通成长游戏
- 玩家是否真的把元石当成生死资源
- 玩家是否会因为药堂债改变路线
- 玩家是否能理解跑路的价值，而不是只把它当失败按钮
- 玩家是否能感到黑市是“门路”而不是“商店”
- 玩家是否能感到传承是“危险跳跃机会”而不是福利箱
- 玩家是否能区分 `canon_strict` 和 `sandbox_if`
- 玩家是否会在关系、物资、修行、地图、因果之间频繁来回查账
- 玩家是否会觉得账本界面信息多但有秩序

## 风险与防漂移条款

### 1. 纵切最容易漂移的方向

- 漂成剧情向视觉小说
- 漂成轻量重开小游戏
- 漂成普通学院成长游戏
- 漂成只剩数值表的硬核模拟器
- 漂成黑市购物模拟

### 2. 纠偏原则

- 任何新增内容，如果不能强化“青茅山秩序下求活”，优先砍掉
- 任何新增系统，如果不能在首个 60-120 分钟里产生真实决策价值，优先延后
- 任何新增自由度，如果会显著稀释债务、暴露、锚点和路线压力，优先压回去
