## 文档角色

本文冻结 Sprint 2.2 发现的一个原著设定边界：首发 S0 不应让玩家在未开窍状态下进行月光修行。开局应落在开窍大典刚结束或大典余波中，让玩家立即进入家族秩序、空窍、元石和修行压力的主线，而不是在没有空窍的状态下先玩一段与修行系统冲突的自由行动。

## 冻结规则

- S0 默认开局语义为“开窍大典刚落定，玩家空窍已开，但根基未稳”。
- 规则状态必须保留 `aperture_opened` 或等价字段，作为凡人修行行动的前置门槛。
- `cultivate`、月光修行痕迹、真元/元石消耗等修行类行动必须依赖空窍已开。
- 如果测试或未来剧情制造 `aperture_opened == false` 的状态，修行行动必须被拒绝或显示明确禁用原因：空窍未开，不能修行。
- 开窍大典本身暂不扩成长剧情段落；Sprint 2.2 只修底层合同与开局一句话，Sprint 3 再决定是否把大典写成完整开场锚点。

## UI 与账本要求

- 主状态条必须显示空窍状态，S0 正常开局显示“空窍：已开窍”。
- 正文开局必须让玩家知道自己不是“未开窍凡人”，而是刚刚进入可修行的家族秩序。
- Build/修行页继续保留本命蛊、核心蛊、辅助蛊的区分；空窍状态不得和本命蛊混写。

## 验收标准

- 新开局投影显示“空窍：已开窍”。
- 新开局正文提到开窍大典或其余波。
- 人工构造未开窍状态时，修行行动不可执行，并给出中文原因。
- 该合同不引入升仙、仙窍、炼化、升转或完整杀招系统。
