## 文档角色

本文档负责定义首发阶段的危险遭遇、短促决断、跑路价值与重创可续后果，回答以下问题：

- 一次遭遇战在本作里到底是什么
- 冲突如何消耗窗口、AP、资源、暴露与债务
- 跑路、拖延、换场、弃子、求饶为什么是核心生存技
- 为什么大多数失败不应立刻死亡，而应转成更糟但可续的局面

本文档不追求复杂战棋系统，而是把冲突写成服务主循环的高压节点。

## 实现前公共接口问题

- 该系统的主循环是什么，最小操作单位是什么
- 它消耗哪些：时间、AP、资源、暴露、债务、人情、空间占用
- 它产出哪些：成长、材料、资格、部署能力、交易能力、关系变化
- 它依赖哪些上位状态：修为、空间、节点、路线、Build、资产所有权
- 它的失败形态是什么：受损、拖慢、欠账、暴露、路线断裂、资产损失
- 它在 `canon_strict` 与 `sandbox_if` 下哪里不同
- 它要在账本 UI 上最少显示哪些信息
- 它如何与首发系统兼容，而不是推翻首发系统

## 设计目标

### 1. 战斗必须短促决断

- 冲突不是长回合拉锯
- 首发遭遇更像数次危险判断压在一个窗口内

### 2. 战前准备高于战中微操

- 侦查、时段、路线、伤势、债务、暴露、节点位置都应显著影响结果
- 玩家不应通过纯战中操作把所有前置失误洗掉

### 3. 跑路必须是高价值按钮

- 跑路不等于认输
- 跑路是把“立刻输更多”换成“留下后患但活着”

### 4. 失败优先转为重创可续

- 首发大多数失败不直接删档
- 常见后果应是伤势、资源损失、底牌暴露、债务加深、名声变差、路线受阻

## 继承约束

- 单个自由窗口基线 `2 AP`
- 冲突依附于节点、时段与窗口，不自成独立长局
- 跑路应在多数危险遭遇中有效，且经常优于硬打
- 受伤恢复必须回到药堂、落脚点、灰路与经济系统中结算
- 暴露与债务必须能反向塑造后续遭遇强度

## 系统定位

### 1. 遭遇不是独立模式

本作首发的遭遇战不是单独切出去的“战斗模式游戏”，而是主循环中的高压分支。

它必须和以下系统直接耦合：

- 节点地图
- 时间/AP
- 暴露
- 经济与恢复
- 路线与 Build

### 2. 最小操作单位

首发遭遇的最小操作单位不是“棋盘上一格”，而是一次 `决断动作`。

决断动作典型包括：

- 试探
- 硬压
- 守住
- 拖延
- 跑路
- 求饶
- 嫁祸
- 弃子保身

## 首发遭遇主循环

一次完整遭遇建议按以下顺序处理：

1. 遭遇触发
2. 给出部分可靠、部分未知的局面信息
3. 进入开局判断
4. 玩家选择第一次决断动作
5. 结算资源、伤势、暴露、位置与局面变化
6. 如局面未结束，进入第二次关键决断
7. 在 `压住 / 脱身 / 受创后逃 / 崩坏` 中落结果
8. 将后果写回主循环

首发不建议把一场遭遇拖成超过 `2-4` 次高价值决断。

## 遭遇输入

一次遭遇至少由以下输入决定：

- 当前节点类型
- 当前时段
- 当前可用 AP
- 当前伤势
- 当前暴露状态
- 当前债务或追索状态
- 当前 Build 倾向与蛊组合
- 事前侦查质量
- 是否有人情、关系或在场旁观者

## 遭遇类型

### 1. 公开秩序型遭遇

典型例子：

- 学堂公开对练
- 被人当众压一头
- 制度内边界试探

特点：

- 暴力致死风险较低
- 名声、地位、比较与求活路线倾向后果更重

### 2. 边路勒索型遭遇

典型例子：

- 巷道堵人
- 试探性勒索
- 黑路门口的探底

特点：

- 跑路、换场、求饶、反咬都很重要
- 资源损失和底牌暴露比直接死亡更常见

### 3. 外缘伏击型遭遇

典型例子：

- 山野误判
- 抢夺
- 埋伏

特点：

- 受伤、掉资源、被迫断线的概率高
- 是首发最适合验证重创可续的遭遇类型

## AP 与冲突

### 1. 遭遇不重置 AP

- 遭遇发生时，默认继承当前窗口剩余 AP
- 玩家如果因为远距离移动或错误时段到达只剩 `1 AP`，就必须吃下这个代价

### 2. 遭遇常见消耗

- `0 AP`：观察、回看、纯信息判断
- `1 AP`：短决断，例如试探、拖延、低成本逃离、轻压
- `2 AP`：重决断，例如硬拼、强突、强追、付大代价反打

### 3. 冲突不能抹掉窗口压力

首发不允许：

- 一进战斗就获得额外专属行动条
- 打赢后等于没花窗口代价
- 跑掉后毫无时间或暴露后果

## 信息与侦查

### 1. 默认局面不是全明

遭遇前后都应保持：

- 一部分信息可靠
- 一部分信息未知
- 一部分信息可能误导

### 2. 侦查必须有价值

成功侦查可带来以下一项或多项：

- 先手判断更准
- 跑路方向更明确
- 更知道对方图什么
- 避开最糟的应对方式

### 3. 失败经验也应有价值

即使第一次吃亏，账本也应记录：

- 哪类点容易出事
- 哪个时段更风紧
- 哪类人会追哪种漏洞

## 跑路系统

### 1. 跑路是核心生存技

首发跑路至少要支持以下几类意图：

- 立刻脱战
- 拖到下一时段或下一地点
- 付资源换脱身
- 付名声或人情换脱身
- 弃掉一部分东西换命

### 2. 跑路的常见代价

- 掉资源
- 暴露上升
- 欠下人情
- 更深伤势
- 门路被记住
- 下一回合行动受限

### 3. 跑路的成功意义

只要玩家保住了：

- 人
- 主 Build 核心
- 后续翻身资格

即使当下亏了，也应被系统视为“有效求活”，而不是失败动画。

## 重创可续

### 1. 首发重创状态

首发建议将战后坏状态优先写成：

- 轻伤可拖
- 重伤压窗
- 核心资源受损
- 路线机会被挤压
- 债务或暴露被放大

### 2. 重创的主要后果

- 下一窗口 AP 被压缩
- 中远距离移动更危险
- 药堂或灰路恢复价格更差
- 某些公开路线不再适合硬走
- 对方更容易继续盯上你

### 3. 何时才适合直接死亡

首发直接死亡更适合出现于：

- 外缘高危点的严重误判
- 已经高伤仍连续硬顶
- 明知无路仍继续深踩灰路

而不是常规试错或第一轮吃亏。

## 战败后果

首发战败后最常见后果应是：

- 被榨干后放生
- 资源被夺
- 底牌暴露
- 伤势加深
- 被迫站队
- 欠下更脏的账

这比“直接 Game Over”更符合项目基调。

## Build 与遭遇的关系

### 1. Build 必须影响应对风格

不同 Build 倾向至少应影响：

- 你更适合硬压还是拖延
- 你能否靠侦查、月光、药理、边路门道撑过一波
- 你在失败时更容易丢什么

### 2. 首发不要求复杂战技树

首发遭遇里不要求完整杀招、阵和仙蛊屋玩法。

但必须为后续兼容预留接口：

- 决断动作可被未来杀招或阵强化
- 位置与场域逻辑可被未来阵与仙蛊屋接管
- 资源与部署消耗不与首发 AP 逻辑冲突

## 模式差异

### canon_strict

- 更少不讲证据的离谱脱身法
- 原著人物与秩序边界更硬
- 跑路可以有效，但代价更清楚

### sandbox_if

- 允许更激进的偏门求生
- 允许更早通过灰路、关系偏航或边路手段脱局
- 但仍不能抹掉伤势、暴露、债务与窗口代价

## 账本 UI 最低要求

遭遇相关界面至少要能显示：

- 当前遭遇类型
- 当前窗口与剩余 AP
- 当前伤势与暴露
- 已知风险与未知风险
- 当前可选决断动作
- 跑路或拖延的大致代价
- 战后最可能写回主循环的坏后果

## 验收标准

- 玩家在多数危险遭遇中都能把跑路视为有效选择
- 玩家至少一次因为“到达时只剩 1 AP”而改变冲突应对
- 玩家至少一次在战败后进入重创可续，而非直接删档
- 玩家能感到侦查和时段选择真实影响了冲突结局
- 玩家能感到公开遭遇、边路遭遇、外缘遭遇的代价结构不同

## 失败信号

- 战斗一进入就像另一款游戏
- 跑路永远是差选项
- 打输后几乎只会直接死
- 冲突与暴露、债务、恢复完全脱钩
- 首发遭遇靠堆战中技能解决，而不是靠准备和判断
