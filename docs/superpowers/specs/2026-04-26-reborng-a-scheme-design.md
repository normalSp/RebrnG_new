## 文档角色

本文件同时承担两层职责：

- 产品宪法：规定整个项目长期不变的设计方向、边界与承诺。
- 首发阶段合同：规定青茅山首发版本必须兑现的内容、节奏与约束。

本文件不讨论语言、框架、工程实现细节，只解决方案设计。

## Summary

- 产品定位：PC 优先、中长局、单核心角色、冷峻主导的文字策略 RPG，不做轻量人生重开小游戏，不做纯视觉小说。
- 世界结构：前中期严格受原著锚点约束，后期逐步开放高阶 IF。
- 总愿景：设计覆盖凡人到尊者级 IF；十转仅作为极后期超高自由扩展预留，不作为 `canon_strict` 的默认承诺目标，也不可反向决定前中期 canon 语义。
- 首发范围：只做青茅山核心圈，把一转前后的制度压迫、资源争夺、黑市门路、传承残线和月光修行做深。
- 模式结构：默认同时开放 `canon_strict` 与 `sandbox_if`，不额外提供传统难度档。
- 单局目标：长期目标是同一角色、同一存档可贯通多个阶段；但各阶段允许展开新的系统层，不要求永远使用完全同构的一套操作壳。

## 产品宪法

### 1. 核心身份

- 本作是铁锚沙盒，不是职业树驱动的数值爬塔，也不是轻叙事抽卡式重开游戏。
- 玩家始终以单主角视角进入世界，但后期允许管理越来越多的外部资产、据点、交易、部署与空窍内资源。
- 本作主基调是冷峻、克制、利益导向、因果导向，不写现代吐槽风，不把原著世界软化成安全的成长童话。

### 2. 单局连续性目标

- 项目长期目标不是“凡人篇一款游戏、蛊仙篇另一款游戏”。
- 长期目标是同一角色可从凡人起步，跨地图、跨阶段、跨生态位持续演化。
- 前期建立的出身、债务、残伤、名声、仇怨、传承线索、关系选择、修行倾向，后期都应尽量延续，不轻易清零。
- 阶段切换可以增加新页面、新系统层、新结算逻辑，但不应推翻前期角色身份。

### 3. 原著锚点原则

- `canon_strict` 下，原著硬事实、关键人物核心结果、重大锚点顺序必须受到保护。
- `sandbox_if` 下，允许更大的偏移、补完和高阶 IF，但仍需服从世界规则、因果门禁与主线保护。
- 原著核心人物与主角维持危险邻近关系，可以感到余波、缝隙、机会和压力，但不轻易做成陪跑队友或传统主线绑定对象。

### 4. 隐藏变量原则

- 天意关注、锚点压力、时间线偏移等高层世界变量长期存在。
- 这些变量不应被粗暴显示成明面数值条。
- 它们必须通过梦兆、异样巧合、风声、人物反应、事件偏斜、资源波动和因果回音来呈现。

## 阶段实现地图

### S0：青茅山凡人到一转

- 家族、学堂、月光、功绩、药堂、黑市残线、传承残线。
- 目标是做深“制度内求活”和“制度边缘冒险”的第一轮闭环。
- 阶段成功定义为“站稳一转根基”。

### S1：一转至五转的区域生存与势力依附

- 扩展更多区域起始包与中层生存圈。
- 强化路线分化、资源依附、势力绑定、迁移风险与跨区压力。
- 仍以人身求活和局部势力秩序为主，不直接跳到蛊仙经营逻辑。

### S2：升仙与仙窍初建

- 引入升仙门槛、仙窍形成、修行结构质变。
- 前期的修行、债务、传承、关系与资源积累要在此处兑现分化。
- 此阶段开始，角色从单一凡人循环进入更高维度的经营与风险结构。

### S3：蛊仙经营、宝黄天交易、阵与仙蛊屋

- 引入仙窍经营、仙元与仙材、跨域交易、宝黄天挂单、阵与仙蛊屋的部署和使用。
- 这一层不是首发系统的简单放大版，而是新的经营与部署层。
- 但它必须继承前期世界逻辑、账本体系与因果延续。

### S4：尊者级 IF 末段

- 允许与原著高层力量同层竞争。
- 不默认爽文化超模，不默认必胜，不取消世界规则的反噬。
- 只在高自由 IF 框架下逐步开放。

## Core Design

### 1. 单局主循环

基础循环：

`节点移动 -> 场景/行动选择 -> 资源与关系结算 -> 风险暴露变化 -> 锚点与世界变量回算 -> 下一时段`

约束说明：

- 这是全游戏的基础循环，不是所有阶段都必须使用完全相同的交互壳。
- 后续仙窍经营、宝黄天交易、阵地部署、仙蛊屋激活等系统，可以在同一账本体系下展开为新子循环。
- 新子循环必须回接到总循环，而不是变成另一款完全无关的游戏。

### 2. 时间系统

- 平时按可见行动资源推进。
- 关键剧情按章节窗口强制触发。
- 长修行、养伤、潜伏、往返与恢复，允许使用日/月级跳时。
- 时间不是纯装饰，而是资源、窗口、恢复、曝光和锚点压力的共同驱动器。

### 3. 行动资源

- 正式版显式显示 AP / 行动点。
- 必须明确标注为玩法抽象，不伪装成原著术语。
- AP 只负责表达“这一阶段可支配注意力与行动额度”，不承担全部世界模拟。

### 4. 存档规则

- 采用阶段存档，支持中长局分次游玩。
- 不允许高频回退每个选择。
- 允许玩家保住中长局投入，但不允许通过读档把因果风险打成空壳。

### 5. 单局结算

- 以死亡为主要终止方式。
- 辅以少量阶段性成功结算。
- 首发阶段主成功定义为“站稳一转根基”，不是“赢下青茅山”。

### 6. 失败形态

- 大量失败不会立刻死亡，而是转化为重伤、残蛊、债务、名声崩坏、路线断裂、被迫站队。
- 战败以“被榨干后放生”与“带伤活下去”作为常见结果。
- 少量情境允许直接死亡，以维持世界危险度。

## 战斗与生存

### 1. 战斗形态

- 早期战斗采用短促决断型遭遇战。
- 重点是战前准备、情报、地形、时机、携带蛊与退路，而不是战中高频微操。
- 中后期允许扩展为部署型、阵地型、复合型冲突，但仍坚持战前准备高于战中微操。

### 2. 战前信息

- 默认部分可靠、部分未知。
- 侦查、尾随、买消息、关系线、失败经验都必须提供真实价值。
- “不知道”本身必须是可设计、可感知、可管理的风险，不是单纯随机坑杀。

### 3. 撤退机制

- 跑路、拖延、换场、求饶、嫁祸、弃子都属于核心生存技。
- 撤退不应是低价值按钮，而应是大量危险遭遇中的合理上策。
- 撤退的代价应真实存在，包括丢资源、丢脸面、暴露底牌、被记仇、欠人情。

## Build 与路线生态

### 1. Build 定义

本作不做开局选职业，采用机会驱动形成。

Build 由七层共同构成：

- 求活路线层：月光、功绩、药堂、黑市、传承等首发与中层活法。
- 主修流派 / 道痕层：中后期长期成长、兼修压力与高阶 Build 主轴的原著保留钩子。
- 核心蛊：真正改变局内决策结构的蛊虫或组合核心。
- 配套系统：杀招、蛊方、阵、部署物、后期仙蛊屋部件等。
- 关系与债务结构：谁保你、谁盯你、你欠谁、谁可被利用。
- 资源来源：功绩、交易、药堂、人情、传承、黑市、势力供养等。
- 维持与喂养负担：蛊虫喂养、供养链、仙窍维护与长期消耗。

### 2. 路线原则

- 路线不是永久职业树。
- 路线是 early-game 生存生态位，是由制度、机会、风险和资源逼出来的求活方向。
- 后期允许多路线交织、断裂、转向、复合，不强求“一条路线玩到底”。

### 3. 首发路线结构

- 主路线：月光、功绩、黑市、传承
- 半主路线：药堂
- 支撑系统：体魄训练

### 4. 路线功能

- 月光：制度内修行、学堂竞争、正面成长
- 功绩：制度内资源争夺、稳健积累、受审计的生存术
- 黑市：制度边缘的危险翻身法
- 传承：制度外的高风险跳跃机会
- 药堂：疗伤、欠债、材料、毒、人情交换与恢复节奏的粘合层
- 体魄训练：对抗受伤、硬碰、跑路、反杀和白刃风险的底层支撑

## 首发阶段合同：青茅山

### 1. 主舞台

- 首发主舞台是家族 + 学堂。
- 山野、黑市、传承属于高风险分流，不是前期主厅。

### 2. 安全区

- 前期真正安全空间极小。
- 只有少数地点和时段勉强安全。
- 大多数节点只是“暂时没那么危险”，而不是无条件安全。

### 3. 个人据点

- 首发只提供简陋落脚点。
- 可藏少量物资、短暂调息、压一压暴露。
- 不构成稳定基地，也不提前做家园经营。

### 4. 移动成本

- 以风险暴露为主，附带时间与窗口损失。
- 风险密度必须贴原著社会逻辑，不夸张成常驻全图追杀。

### 5. 黑市表达

- 黑市是隐藏网络，不是开局可见商店。
- “知道门路”本身就是资源。
- 首发前期黑市的价值重在获得机会、消息、替代路径与危险交易，不是稳定购物。

### 6. 传承表达

- 以传闻、残线、半真半假线索为主。
- 真正入口稀少且危险。
- 传承不是前期福利房，而是制度外的高风险跃迁可能。

### 7. 脱离家族秩序

- 定义为中后段资格。
- 前期主要是在秩序缝隙里求活，而不是开局单飞。

### 8. 师长关系

- 主基调是庇护与利用并存。
- 允许极少数例外，但不做传统导师制。

### 9. 原著核心人物

- 与方源等 guarded NPC 保持危险邻近关系。
- 能感到存在、余波、压力和机会，但不轻易深度绑定。
- 首发前期不得直接改写方源等核心人物的硬事实。

## 模式结构

### canon_strict

- 只读取 `canon_explicit` 与 `canon_inferred` 证据。
- 缺证据内容不能硬塞进关键事件、交易结果、奖励数值和主线因果。
- 缺证据内容只能延后、转化为风声、留作 IF、或被明确禁用。

### sandbox_if

- 允许更高比例的玩法补完、原创侧线和锚点偏移。
- 仍需经过主线保护、世界规则门禁和因果校验。
- 不是“想写什么就写什么”的放飞模式。

### 双模式关系

- 两模式共享同一世界规则与大部分内容骨架。
- 差异主要在证据门禁、锚点容差、原创允许度与世界反噬强度。
- 不应演化成两套互不相干的游戏。

## 内容与运行时设计原则

### 1. 内容哲学

- 系统生成主导，但质量下限必须接近 authored 内容。
- 系统生成负责规模、分化、承接与重玩。
- 策展与证据治理负责世界质感、风格稳定和逻辑下限。

### 2. 生成策略

- 采用离线 + 实时混合。
- 离线层负责批量生成、筛选、验真、打标签、策展入库。
- 实时层只负责局部变体、上下文化、因果承接、语气统一和状态映射。
- 关键场景、关键锚点、重大转折不得完全现场裸生。

### 3. 原创占比

- 首发按中占比控制。
- 主轴和关键因果必须紧贴原著或高可信推断。
- 可玩密度允许由原创侧线、衍生关系、风险变体与局部遭遇撑起。

### 4. 亲密关系

- 只做弱存在。
- 允许少数高风险深关系。
- 不做恋爱养成主系统，不让情感关系吞掉生存、资源与因果主轴。

## 长期兼容条款

### 1. 基础循环与子循环

- 所有阶段共享同一总循环，但不要求所有玩法都塞进同一种局部操作壳。
- 仙窍经营、宝黄天交易、阵地部署、仙蛊屋激活、组织委托等都可以展开为子循环。
- 子循环必须回接总账本、总因果、总风险与总角色。

### 2. 经济分层

前期经济不等于全游戏经济。

经济至少分三层理解：

- 凡人层：元石、材料、功绩、债务、人情、配给、黑市门路
- 中层势力层：供应、路线、护送、仓储、账务、交换权、组织依附
- 蛊仙层：仙元、仙材、仙窍产出、跨域交易信用、宝黄天挂单、长期供需关系

设计约束：

- 黑市不能被误写成未来宝黄天的简化版。
- 药堂债、人情债、组织债应与蛊仙期的交易信用和挂单信誉分层处理。

### 3. 单主角与多资产操作

- 本作始终是单主角视角。
- 但后期允许主角管理多层资产与外部载体。

这些资产至少包括：

- 随身资源
- 藏点与据点库存
- 空窍内资源与驻留蛊
- 可部署之阵、场域与防线
- 仙蛊屋部件、完成体与使用状态
- 交易挂单、交换承诺、外部委托物

设计约束：

- 单主角不等于“永远只管理自己包里那几样东西”。

### 4. Build 的后期扩展

- 首发 Build 主要由蛊、杀招雏形、蛊方、人情与资源线构成。
- 后期 Build 必须自然扩展到阵、仙蛊屋、仙窍经营结构、交易网络与势力绑定。
- 因此路线不能被设计成单薄的“加点树”或“学院分支”。
- `求活路线`、`主修流派`、`道痕演进` 与 `蛊虫喂养和维护负担` 不能被塞进同一个长期字段。

### 5. 空间分层

空间至少分四层理解：

- 节点层：当前所在地点、节点移动、潜入、撤离、藏身
- 区域层：青茅山、商家城外围、村寨、山线、路网
- 隐秘层：黑市门路、传承入口、暗线空间、隐藏接触点
- 内部层：空窍、后期仙窍及其经营空间

设计约束：

- `当前地点` 不是全世界空间系统的全部。
- 空窍与仙窍不是普通地图节点，它们是角色内生的独立系统层。

### 6. 账本式母界面

- 账本式界面是全项目的母表达，不是首发一屏塞满一切的死布局。
- 正文、因果账本、关系局势、空窍/修行状态、节点地图、物资债务、线索传闻都应归属于同一账本体系。
- 允许按阶段展开为不同账页、侧页、子视图、交易板、部署页、经营页。

设计约束：

- 后期新系统必须“像同一本账册的新页”，而不是另一套完全陌生的 UI 语言。

## Launch Scope 细则

### 1. 经济压力

- 6-8 回合内出现明显的元石紧张、欠债压力和资源取舍。
- 黑市在未解锁前不可见，解锁后能显著提升机会但抬高暴露风险。
- 药堂债能真实锁选项、改价格、带来追索。

### 2. 路线分化

- 月光、功绩、黑市、传承四主路线在前 8 回合内出现可感知分化。
- 药堂能作为半主路线稳定咬合至少两条主路线。
- 体魄训练能在跑路、受伤恢复、白刃冲突和高压劳作里体现稳定价值。

### 3. 遭遇战

- 玩家可通过侦查拿到部分真实情报，但仍保留盲区。
- 跑路在多数危险遭遇中是有效且常常更优的生存选择。
- 战败后常见进入“重创可续”状态，而非总是立即死亡。

### 4. 锚点保护

- 首发前期不能直接改写方源等核心人物的硬事实。
- 高层隐藏变量只给征兆，不直出数值条。

### 5. 单局结算

- 多数局以死亡结束。
- 少数局可通过“站稳一转根基”触发阶段性成功。

### 6. 模式差异

- `canon_strict` 必须拒绝缺证据内容。
- `sandbox_if` 允许补完内容，但仍保留主线门禁和世界逻辑。

## 非目标：S0 不承诺项

- 不承诺首发即做完整升仙系统。
- 不承诺首发即做完整仙窍经营。
- 不承诺首发即做宝黄天通用交易盘。
- 不承诺首发即做完整阵地部署与仙蛊屋使用。
- 不承诺首发即做全地图自由流亡式大世界。
