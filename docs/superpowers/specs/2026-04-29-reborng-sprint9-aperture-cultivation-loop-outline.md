## Summary
Sprint 9 把 Sprint 8 已经可见的空窍、资质、元海、真元、小境界推进为可结算玩法。目标不是还原原著精确数学，而是先让玩家理解：资质会影响真元恢复与冲刷效率，月光蛊炼化和修行会消耗真元，修行不是点按钮升级。

本轮只做凡人一转初期轻公式，不做酒虫稳定获得、花酒传承奖励、炼蛊升转、杀招、升仙、仙窍或宝黄天。酒虫与花酒外围残线建议放到 Sprint 10，在修行底座稳定后接入。

## Canon Index References

- `primeval_stone_refill_and_resource_pressure`：元石仍是修行资源压力的一部分。
- `aperture_wall_small_realm_progression`：光膜、水膜、石膜作为小境界推进的事实钩子。

## Key Rules
- `MortalApertureState` 新增 `essence_recovery_per_window` 与 `aperture_wall_progress`。
- 丁 / 丙 / 乙 / 甲自然恢复分别为 `4 / 7 / 11 / 15` 真元，窗口推进时自动恢复并封顶为元海容量。
- 新增行动 `RecoverEssence`，显示为“调息回元”，消耗 `1 AP`，按资质恢复真元，不提高元海上限。
- 炼化月光蛊消耗 `1 AP + 8 真元`，不消耗元石。
- 月光修行消耗 `1 AP + 1 元石 + 10 真元`，并推进空窍壁冲刷进度。
- 丁 / 丙 / 乙 / 甲单次月光修行冲刷进度分别为 `6 / 8 / 11 / 14`。
- 冲刷进度达到 `100` 时，首次小境界跃迁为 `一转初阶 / 光膜 -> 一转中阶 / 水膜`，进度清零。

## UI 与反馈
- 空窍账必须显示：资质、元海容量、当前真元、自然恢复、空窍壁、冲刷进度、小境界。
- 真元不足时，月光修行和炼化必须显示中文禁用原因：`真元不足`。
- 对话流和最近反馈必须能说明本次行动造成的真元、冲刷或小境界变化。
- React 只消费 Rust 投影，不推导真元公式、不决定修行可用性。

## Test Plan
- 开窍后真元初始值等于元海容量，恢复速度按资质生成。
- 未开窍时不能炼化、调息或修行。
- 炼化月光蛊扣 `8 真元`，月光修行扣 `10 真元 + 1 元石`。
- 调息和窗口推进都会恢复真元，但不能超过元海容量。
- 冲刷进度到 `100` 时推进到一转中阶，并更新空窍壁为水膜。
- 存档读回后保留当前真元、恢复速度、冲刷进度、小境界、空窍壁和月光蛊状态。

## Assumptions
- 本轮公式是首发轻量玩法基线，可在后续平衡 Sprint 中调整，但调整必须升级 `RULES_VERSION`。
- `canon_strict` 与 `sandbox_if` 共用同一修行公式；差异只来自开局资质、IF 异象、线索和锚点压力。
- 小境界只开放第一次跃迁到一转中阶；一转高阶、巅峰、升转、杀招、酒虫效率加成均后置。
