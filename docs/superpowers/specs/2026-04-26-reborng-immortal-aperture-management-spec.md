## 文档角色

本文档负责定义升仙后的仙窍经营层，回答以下问题：

- 仙窍在本作里是什么空间
- 仙窍经营的主循环是什么
- 仙窍为什么不能只是“另一个背包”
- 仙窍如何给交易、Build、阵与仙蛊屋提供支撑

## 实现前公共接口问题

- 该系统的主循环是什么，最小操作单位是什么
- 它消耗哪些：时间、AP、资源、暴露、债务、人情、空间占用
- 它产出哪些：成长、材料、资格、部署能力、交易能力、关系变化
- 它依赖哪些上位状态：修为、空间、节点、路线、Build、资产所有权
- 它的失败形态是什么：受损、拖慢、欠账、暴露、路线断裂、资产损失
- 它在 `canon_strict` 与 `sandbox_if` 下哪里不同
- 它要在账本 UI 上最少显示哪些信息
- 它如何与首发系统兼容，而不是推翻首发系统

## 设计目标

### 1. 仙窍必须是独立经营空间

- 仙窍不是扩容背包
- 它是会产出、会消耗、会失衡、会拖垮角色的内部空间

### 2. 经营必须有支撑意义

- 仙窍经营不是单纯种田
- 它必须直接支撑交易、Build、部署和长期生存

### 3. 经营必须有维护成本

- 产出不是白给
- 维护失败必须带来断供、减产、资产损耗或后续系统受限

## 仙窍定位

仙窍是升仙后新增的独立空间层，至少承担以下职责：

- 资源产出空间
- 资产承载空间
- Build 维护空间
- 部署和交易的后勤底座
- 初成福地与后续洞天跃迁等形态的承载基础

## 仙窍经营主循环

仙窍经营建议按以下循环组织：

1. 检查当前仙窍状态
2. 检查当前时间流速与灾劫周期位置
3. 决定本周期重点
4. 分配资源、喂养与维护动作
5. 结算产出、消耗与时间压力
6. 处理异常、断供、超载或灾劫临近
7. 将结果写回主地图、交易与 Build 层

最小操作单位建议为 `经营决策单元`，而不是地块级细碎点击。

## 仙窍输入

经营至少受以下因素影响：

- 当前修为阶段
- 当前仙窍承载状态
- 当前时间流速或等效时间压力
- 当前灾劫周期位置
- 当前福地阶段 / 洞天跃迁结构位
- 当前产出结构
- 当前驻留资产与核心蛊需求
- 当前蛊虫喂养与维护负担
- 当前外部交易压力
- 当前部署系统需求
- 当前势力依附或委托压力

## 仙窍产出

仙窍产出不只包含材料，还应包含：

- 维持 Build 的供给能力
- 支持交易的货源能力
- 支持阵与仙蛊屋的后勤能力
- 支持组织关系或委托的交付能力

## 仙窍负担

仙窍至少应承担以下负担：

- 维护消耗
- 产出波动
- 资产占位
- 过载风险
- 时间流速带来的管理压力
- 灾劫周期压力
- 蛊虫喂养与供养负担
- 特定路线或 Build 的偏食要求

## 必须保留的原著状态钩子

即使本轮不写细公式，仙窍层也必须保留：

- `时间流速 / 等效时间压力`
- `灾劫周期 / 周期性大风险`
- `福地阶段 / 洞天跃迁差异保留口`
- `蛊虫喂养与维护负担`

## 资产容器关系

仙窍是资产容器之一，但与以下容器不同：

- 随身
- 落脚点
- 挂单中
- 部署中
- 组织托管中

仙窍内资产不应默认等于“随时可取随时可战”。

补充说明：
蛊仙经营默认先围绕 `福地阶段` 展开；`洞天` 只作为更后段的空间跃迁位预留，不应在初次升仙后就被压成同级常态。

## 经营重点类型

中后期经营重点建议至少有以下几类：

- 保底稳产
- 冲刺某类产出
- 优先供养核心 Build
- 优先支撑交易链
- 优先支撑阵与仙蛊屋维护

这能保证不同路线的蛊仙不是同一种种田器。

## 经营失败形态

常见失败形态应包括：

- 产出不足
- 维护中断
- 资产损耗
- 某条交易或部署链断供
- 灾劫周期逼近时被迫拆东墙补西墙
- 被迫抽走主地图资源救火

经营失败不应只写成“收入变少一点”。

## 与首发系统的兼容

- 时间窗口逻辑仍保留，只是增加更长经营周期
- Build 仍是主轴，只是新增经营支撑层
- 债务与关系仍有效，只是多了更大体量的维护义务

## 模式差异

### canon_strict

- 仙窍经营更受原著逻辑限制
- 特殊高效产出链更难白给

### sandbox_if

- 允许更激进的偏门经营
- 允许更早尝试高风险高收益的结构

## 账本 UI 最低要求

仙窍页至少要显示：

- 当前经营重点
- 当前主要产出
- 当前主要负担
- 当前时间流速与下一次灾劫压力
- 当前关键断供风险
- 哪些外部系统正在吃仙窍产出

## 验收标准

- 仙窍被定义成独立空间与经营层，而不是大背包
- 仙窍产出能支撑交易、Build 或部署
- 仙窍经营失败会真实影响后续系统
- 仙窍与资产容器分层清楚

## 失败信号

- 仙窍只是数值农场
- 仙窍产出和主 Build 毫无关系
- 仙窍资产与随身资产完全不分
