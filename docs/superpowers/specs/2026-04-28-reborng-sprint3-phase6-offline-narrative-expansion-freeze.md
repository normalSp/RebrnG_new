## 文档角色

本文冻结 Sprint 3 Phase 6 的叙事生产合同。Phase 6 的目标不是接入 runtime AI，也不是扩写完整长局，而是先把“玩家最终读到的文字应该是什么质感”与“DeepSeek 如何作为离线候选工具被约束”定清楚。

后续任何正文扩写、候选文本生成、人工改写或内容入库，都必须继承本文。规则事实仍由 Rust 规则层写入；内容包仍由 YAML 和构建器提供；AI 只允许产出待审候选。

## 核心结论

- 玩家主观体验可以接近豆包式人生重开：长正文、强选择、明确结果、阶段推进。
- 项目底层机制不改变：所有行动仍进入 `ActionCommand` 管线，定量变化仍来自 Rust 规则结算。
- DeepSeek 只能作为离线内容候选生成工具，不得进入玩家点击后的同步主链。
- 候选文本必须经人工审核、证据打标、模式门禁和内容构建校验后，才能进入 `content/s0/**/*.yaml`。
- `canon_strict` 只允许压迫感、资源争夺、制度逻辑和高可信推断；高自由强改写只允许进入 `sandbox_if`，且仍需要代价和锚点保护。

## 叙事体验合同

正文必须服务四件事：
- 处境：玩家当前在何处、面对什么秩序、谁在看、窗口还有多窄。
- 选择：玩家刚做了什么，为什么这件事在蛊界有意义。
- 代价：AP、元石、债务、暴露、伤势、线索、关系或路线如何变化。
- 因果：这次行动把下一步压力推向哪里。

禁止项：
- 空泛 AI 作文，只写氛围、不写具体状态变化。
- 现代吐槽、网文作者旁白、系统爽文口吻。
- 爽文化直给奖励，例如“直接获得完整传承”“被大能无条件看重”。
- 规则事实乱编，例如凭文本生成蛊虫、杀招、元石、传承或锚点改写。
- 用概率和成功率大段直出替代蛊界内的征兆、风声和人物反应。

推荐质感：
- 第二人称，冷峻克制，利益优先，因果清楚。
- 人物反应要带利用、试探、审计、忌惮或冷眼，而不是传统导师式善意。
- 场景细节优先服务行动后果，例如药堂账簿、学堂目光、巷道脚步、黑市门路。
- 文本可以更长，但每段都要推动处境、压力或选择理解。

## 文本体量目标

设置与开窍段落：
- 每个关键设置节点推荐 3-5 段短正文。
- 必须解释出身压力、天赋代价、开窍结果和家族秩序如何影响开局。

普通行动结果：
- 推荐 1-2 段。
- 必须给出明确点击反馈和至少一项账本变化或无变化原因。

遭遇触发与决断反馈：
- 推荐 2-4 段。
- 必须写清遭遇来源、可见风险、玩家决断、直接代价和余波。

阶段收口：
- 推荐 3-5 段。
- 必须总结本局根基、伤势、债务、暴露、线索和后续锚点压力。

线索与风声：
- 推荐 1 段短文本。
- 必须保持半真半假，不把隐藏变量、天意数值或原著硬事实直接摊开。

## 叙事槽位

后续扩写只允许填充这些槽位，不允许绕过规则层生成奖励：
- `setup_paragraph`：人生重开设置正文。
- `opening_rite_paragraph`：开窍大典与空窍已开后的余波。
- `action_result`：普通行动结果。
- `movement_result`：移动、到达和当前位置变化。
- `encounter_trigger`：遭遇触发。
- `encounter_decision`：遭遇决断结果。
- `stage_closure`：阶段收口。
- `rumor_clue`：线索、风声和传闻。
- `blocked_reason`：禁用原因的世界内表达。
- `ledger_delta_summary`：账本变化的可读摘要。

## 离线候选文本接口

离线候选文本不是运行时内容。建议最小字段如下：
- `candidate_id`：候选文本 id。
- `target_content_id`：目标内容 id，例如行动、节点、遭遇或叙事模板 id。
- `target_slot`：叙事槽位。
- `mode`：`canon_strict`、`sandbox_if` 或两者兼容。
- `evidence`：证据等级，沿用内容包证据体系。
- `candidate_text`：候选正文。
- `state_assumptions`：生成文本依赖的可见状态，例如地点、时段、伤势、债务或线索。
- `risk_notes`：原著风险、机制风险或文本风险。
- `review_status`：审核分级。
- `review_notes`：人工审核意见。

候选文本禁止包含：
- DeepSeek API key、模型配置、完整 prompt 或完整 response。
- 未经规则层确认的数值变化。
- 未经内容门禁确认的蛊虫、传承、杀招、NPC命运或原著锚点变化。
- 可直接复制进 runtime 的脚本调用或网络调用。

## 审核分级

`可直接入库`：
- 符合模式许可和证据等级。
- 不新增规则事实。
- 能明确增强行动反馈、压力理解或阶段推进。

`需改写`：
- 方向可用，但文字空泛、过长、过爽、信息不清或口吻不稳。
- 存在轻微原著风险，但可通过删改解决。

`仅 sandbox_if`：
- 涉及强天赋、原著机缘前置、路线大偏移或高自由 IF。
- 仍必须保留代价、暴露、天意压力、锚点保护和资源门槛。

`拒绝`：
- 在 `canon_strict` 中生成稳定本命蛊、完整盗天核心真传、提前夺花酒传承硬成功、方源硬绑定合作、尊者级硬改写或无代价越阶收益。
- 把蛊虫写成普通装备，把杀招写成技能树按钮，把炼蛊写成无风险 crafting。
- 让 AI 文本绕过 Rust 规则层直接决定资源、伤势、债务、蛊虫、传承或结局。

## DeepSeek 离线边界

允许：
- 离线生成候选正文。
- 离线生成同一槽位的多版本文本供人工筛选。
- 离线辅助做语气统一、压缩、改写或错别字检查。
- 离线根据已导出的状态摘要生成“文本建议”，再由人决定是否入库。

禁止：
- 在 `resolve_action` 中调用 DeepSeek。
- 在 Tauri command 中调用 DeepSeek。
- 在 React 点击链路中调用 DeepSeek。
- 在玩家等待下一回合时同步请求 DeepSeek。
- 在存档中保存 prompt、response、模型名、API key 或未审核草稿。
- 把 DeepSeek 输出直接写入 `GameState`、`SaveEnvelope` 或运行时账本。

密钥规则：
- 不得把 DeepSeek key 写入仓库、文档示例、测试、YAML、存档或构建产物。
- 若后续建立离线工具，只能读取本地环境变量，例如 `DEEPSEEK_API_KEY`。
- 本地 `.env.local` 如需使用，必须保持 git ignore，且不得成为运行游戏的必需条件。

## 内容入库流程

1. 导出上下文包：只包含目标 id、模式、证据等级、可见状态、允许事实、禁止事项和目标槽位。
2. 离线生成候选：DeepSeek 只输出候选文本或候选 YAML 草案。
3. 人工初审：检查原著风险、机制风险、模式许可和文本质感。
4. 人工改写：删掉超模奖励、隐藏数值直出、现代口吻和规则乱编。
5. 写入 YAML：只把通过审核的文本写入 `content/s0/**/*.yaml`。
6. 构建校验：运行 `pnpm content:build`，由 Rust 内容构建器校验 id、模式、证据和索引。
7. 回归走查：确认对话流、最近反馈、账本明细同时可读。

## canon_strict 风险红线

`canon_strict` 下禁止：
- 开局稳定获得本命蛊。
- 开局稳定拿到完整盗天核心真传。
- 提前夺取花酒行者传承并硬成功。
- 直接说服、绑定或控制方源。
- 让族中高层无条件投资玩家。
- 缺少代价地获得大量元石、蛊虫、蛊方或杀招。
- 将传承线写成安全奖励通道。
- 将天意、锚点压力或隐藏世界变量数值化直出。

`canon_strict` 下允许：
- 低可信传闻、残线、半真半假的风声。
- 制度内修行压力、学堂比较、药堂债、功绩审计、黑市门路。
- 由原著社会逻辑推断出的试探、勒索、排挤、利用和风险。
- 月光蛊等首发范围内已有证据支撑的低阶内容。

## sandbox_if 许可边界

`sandbox_if` 可以更早触及强 IF 内容，但必须满足：
- 明确标记模式许可。
- 明确代价来源：资源、债务、暴露、伤势、关系、锚点压力或路线断裂。
- 不把 IF 内容反向写成 `canon_strict` 的默认事实。
- 不绕过修为、空窍、蛊虫、喂养、炼化、升转、杀招组成等既有合同。
- 不把高阶尊者级结果提前压到 S0 首发开局。

## Phase 6 验收标准

- 本文档明确 runtime AI 禁止项。
- 本文档明确 DeepSeek 密钥禁止入库。
- 本文档明确候选文本必须人工审核后才能进入 YAML。
- 本文档明确 `canon_strict` 与 `sandbox_if` 的文本边界。
- 本文档明确后续扩写必须通过内容构建器，不允许 AI 直接写运行时状态。
- 本阶段不改变 `ContentManifest.version`。
- 本阶段不改变 `RULES_VERSION`。
- 本阶段不新增 runtime command、AI 设置页、网络调用或运行时 AI provider。

## 归档说明

本文通过后，Sprint 3 后续可以进入两个方向：
- 建立离线候选生成工具，但仍不得接入 runtime。
- 扩写 S0 正文、行动结果、遭遇反馈和阶段收口文本，并按本文进行人工审核与内容构建。

在这两步之前，不应继续扩大高阶系统，也不应把 DeepSeek 直接接到玩家点击后的主链。
