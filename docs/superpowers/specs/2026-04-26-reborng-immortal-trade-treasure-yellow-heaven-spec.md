## 文档角色

本文档负责定义蛊仙阶段的交易层与宝黄天语义，回答以下问题：

- 宝黄天在本作里到底是什么
- 蛊仙交易为什么不能退化成一个高级商店页
- 挂单、信用、供需和仙窍产出如何连成一体
- 它和凡人期黑市、药堂债有什么本质区别

## 实现前公共接口问题

- 该系统的主循环是什么，最小操作单位是什么
- 它消耗哪些：时间、AP、资源、暴露、债务、人情、空间占用
- 它产出哪些：成长、材料、资格、部署能力、交易能力、关系变化
- 它依赖哪些上位状态：修为、空间、节点、路线、Build、资产所有权
- 它的失败形态是什么：受损、拖慢、欠账、暴露、路线断裂、资产损失
- 它在 `canon_strict` 与 `sandbox_if` 下哪里不同
- 它要在账本 UI 上最少显示哪些信息
- 它如何与首发系统兼容，而不是推翻首发系统

## 设计目标

### 1. 宝黄天不是大号黑市

- 宝黄天不是黑市升级版
- 它是蛊仙层的特殊交易空间，核心在接入媒介、长期供需、挂单资格与后果性信用

### 2. 交易必须依赖供货与信用

- 没有稳定产出和挂单资格，就不能稳做交易
- 交易应和仙窍经营紧密绑定

### 3. 交易必须带等待与风险

- 交易不应总是即时买入卖出
- 需要挂单、等待、履约、被放鸽子、被卡条件的空间

## 交易层级

中后期交易至少分为：

- 地方交易
- 组织内部调配
- 跨图交换
- 宝黄天挂单交易

## 宝黄天定位

宝黄天至少承担以下功能：

- 挂单空间
- 接入媒介与神念-意志交互层
- 挂单资格层
- 后果性信用层
- 跨域供需交换空间
- 高价值信息和稀缺货物流通空间

## 蛊仙交易主循环

建议按以下循环组织：

1. 准备可交易资产
2. 选择交易渠道
3. 挂单或发起交换
4. 等待、谈条件或履约
5. 结算收益、成本、信誉与后续风险

最小操作单位建议为 `一次交易决策` 或 `一次挂单决策`。

## 交易输入

- 当前仙窍产出结构
- 当前接入媒介状态
- 当前神念 / 意志接入负担
- 当前交易资格与信用
- 当前交易能见度 / 价值显露状态
- 当前稀缺需求
- 当前地区和空间可达性
- 当前组织或关系支持
- 当前部署系统的资源占用

## 交易产出

交易至少应产出：

- 资源调剂
- 供需链补齐
- Build 支撑
- 长期信用
- 高价值机会

## 交易成本

交易成本不只包括资源，还包括：

- 时间等待
- 履约压力
- 信用损耗风险
- 被截断供给的风险
- 对外暴露交易习惯的风险

## 黑市与宝黄天的边界

### 黑市

- 边路门路
- 灰色机会
- 首发与凡人期危险翻身法

### 宝黄天

- 蛊仙层挂单与供需空间
- 需要更高结构支撑
- 依赖仙窍、接入媒介、挂单资格和长线交易能力

禁止把黑市的设计原样拉长成宝黄天。

## 信用层

蛊仙交易至少应有独立信用层，用来表示交易后果与资格后续，而不是宝黄天的唯一核心定义：

- 你是不是可靠卖家
- 你是不是总断供
- 你是否老拖单
- 你是否有足够资格承接更重交易

它不能与药堂债或人情债混写。

## 必须保留的原著状态钩子

即使本轮不钉死具体媒介物，宝黄天层也必须保留：

- `接入媒介状态`
- `通过通天类媒介投送神念 / 意志的交互语义`
- `交易能见度 / 价值显露状态`
- `挂单资格与特殊空间门槛`
- `信用作为后果层与资格层之一，而不是唯一主语义`

## 挂单逻辑

挂单至少要回答：

- 货从哪里来
- 在哪种空间里挂
- 挂了多久
- 谁能接
- 没人接或接了毁约怎么办

## 模式差异

### canon_strict

- 宝黄天内容更受原著逻辑约束
- 稀缺高价值交易更谨慎

### sandbox_if

- 允许更激进的交易实验
- 允许更早通过灰线触达部分高风险资源

## 账本 UI 最低要求

交易页至少要显示：

- 当前主要供货来源
- 当前接入媒介状态
- 当前主要缺口
- 当前挂单状态
- 当前信用状态
- 当前交易能见度与门槛
- 当前交易渠道与空间

## 验收标准

- 宝黄天被定义成接入媒介、神念-意志交互、挂单资格与后果性信用共同构成的特殊交易空间，而不是高级商店
- 蛊仙交易与仙窍产出绑定
- 黑市与宝黄天边界清楚
- 交易失败会带来供需或信用后果

## 失败信号

- 宝黄天就是商城
- 没有供给也能无限买卖
- 信用层缺失
- 黑市与宝黄天语义混为一谈
