## 文档角色

本文档负责冻结 Sprint 1 的内容体量、可玩性目标、详细内容表和验收剧本。

它不是新的产品总纲，也不是正式版完整青茅山长局设计。它只回答一个问题：

在 Sprint 0 已经跑通 8 个自由窗口、Rust 行动管线、账本 UI、存档和性能验收之后，下一轮如何把这个最小闭环打磨成“像游戏”的首发纵切。

## 核心结论

Sprint 1 先打磨现有 8 个自由窗口，不扩大局长，不横向扩高阶系统。

本轮冻结以下取舍：

- 范围：`8 自由窗口可玩性打磨`
- 冻结颗粒：`详细内容表`
- 蛊虫深度：`状态展示钩子`
- 主目标：让月光、功绩、药堂、黑市、传承五条入口在 8 窗口内形成真实取舍
- 禁止目标：不做 60-120 分钟完整纵切，不做完整炼化 / 升转 / 杀招系统，不做升仙、仙窍、宝黄天

## Sprint 1 成功定义

Sprint 1 通过必须同时满足：

- 玩家能自然跑完 8 个自由窗口，并在不查测试脚本的情况下理解当前压力来源。
- 至少出现两次真实取舍，来源可以是 AP、元石、债务、暴露、伤势、时段或路线机会。
- 月光、功绩、药堂、黑市、传承五条线都不只是标签，而是有不同的风险结构和收益节奏。
- 黑市仍是隐藏门路，不能变成开局可见商店。
- 传承仍是半真半假残线，不能变成稳定高回报任务链。
- 跑路、恢复、侦查、等待都必须有实际价值，不能只是陪衬按钮。
- 阶段收口能区分“站稳一转根基”“重创可续”“根基不足待判”。
- UI 能解释为什么一个行动可用、不可用、值得做或危险，而不是只显示按钮。

## 范围边界

### 本轮必须做

- 扩展 S0 内容包，补足 8 个可感知节点。
- 给五条路线各补至少 2 个入口级内容点。
- 把危险遭遇从 1 类扩到至少 3 类。
- 增加风声、线索、误导和禁用原因，让玩家能理解风险。
- 打磨账本 UI 的行动反馈、后果反馈和阶段收口展示。
- 增加 Sprint 1 可玩性验收脚本。

### 本轮不做

- 不扩成完整 60-120 分钟纵切。
- 不做完整自由开局。
- 不做完整多 NPC 关系模拟。
- 不做完整黑市商店和通用价格盘。
- 不做完整药堂治疗清单。
- 不做完整蛊虫图鉴、蛊方、炼化、升转、杀招库。
- 不做阵、仙蛊屋、升仙、仙窍经营、宝黄天。
- 不接 runtime AI。
- 不处理 `apps/desktop/src/assets/` 与 `docs/art/` 未跟踪素材目录。

## 叙事模板表

Sprint 1 叙事模板必须覆盖：

| 模板类别 | 数量下限 | 用途 |
| --- | --- | --- |
| 节点进入正文 | 8 | 每个可感知节点至少 1 条 |
| 路线行动反馈 | 10 | 五条线各至少 2 条 |
| 禁用原因文案 | 10 | AP 不足、时段不对、线索未解锁、资源不足、伤势限制等 |
| 遭遇触发 / 决断 | 9 | 3 类遭遇各至少触发、跑路 / 退让、硬顶 / 对抗 |
| 阶段收口 | 3 | 站稳一转根基、重创可续、根基不足待判 |

所有运行时可见正文必须来自内容包模板、账本投影或 Rust 本地 fallback。不得接 runtime AI。

## 蛊虫、炼化、杀招展示边界

Sprint 1 只允许以下层级：

- 显示 `本命蛊：未建立`，不开放获取、绑定、改命或替换。
- 显示 `核心蛊候选：月光蛊线索 / 月光蛊方向` 这类状态钩子，不把蛊虫写成普通装备。
- 显示 `炼化压力：未开放 / 需要时机、资源和空窍承载` 这类占位，不开放完整炼化归属操作。
- 显示 `喂养维护压力：暂无实物蛊虫压力 / 未来会消耗资源` 这类占位，不做自由喂养系统。
- 显示 `杀招雏形：未成型` 或 `月光运用仍停留在基础修行`，不开放杀招按钮、技能树或多蛊组合。

canon_strict 下，具体蛊名、蛊方、杀招和关键奖励必须来自明确证据或高可信推断。

sandbox_if 可以出现原创补完线索，但必须带模式许可和世界规则门禁。

## UI 可玩性打磨表

| UI 区域 | Sprint 1 必须改善 | 禁止事项 |
| --- | --- | --- |
| 顶部状态条 | 常驻显示时段、窗口、AP、地点、暴露、债务、伤势、遭遇 | 不隐藏关键压力 |
| 行动列表 | 从 Rust 投影生成，显示中文短标签、可见代价、风险提示 | 不在 React 硬编码规则真相 |
| 禁用行动 | 显示禁用原因，例如线索未解锁、时段不对、资源不足 | 不让按钮消失到玩家无法理解 |
| 因果账 | 记录行动、代价、后续追索原因 | 不直接渲染英文 fallback |
| Build / 修行页 | 显示求活路线、月光痕迹、本命蛊、核心蛊、辅助蛊、喂养维护压力 | 不把本命蛊和核心蛊混字段 |
| 线索页 | 显示风声、验真状态、误导风险 | 不把隐藏变量原值显示给玩家 |
| 阶段收口 | 显示成功 / 重创 / 待判原因 | 不在 UI 临时判断结局 |

## 验收剧本冻结

### 剧本 A：月光根基成功

目标：证明制度内修行能形成阶段成功，但需要元石和窗口取舍。

必须经历：
- 至少 2 次月光修行
- 至少 1 次因元石或 AP 放弃其他行动
- 进入锚点后显示 `站稳一转根基`

### 剧本 B：黑市跑路存活

目标：证明黑市不是商店，跑路是核心生存技。

必须经历：
- 先通过风声或侦查解锁黑市
- 深夜进入黑市暗口
- 触发危险遭遇
- 选择跑路后存活但增加暴露或断线风险

### 剧本 C：药堂债务恢复

目标：证明恢复不是免费回血。

必须经历：
- 受到轻伤或重伤
- 进入药堂恢复
- 降低伤势
- 增加药堂债或人情债
- 后续至少 1 个行动受到债务压力提示

### 剧本 D：传承诱惑撤退

目标：证明传承残线是高风险诱惑，不是稳定福利。

必须经历：
- 获得半真半假传承风声
- 进入或接近传承残线
- 出现误导、暴露或高风险提示
- 玩家选择撤退或放弃，保留后续悬念
