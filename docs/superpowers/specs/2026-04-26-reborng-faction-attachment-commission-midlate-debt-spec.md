## 文档角色

本文档负责定义中后期的势力依附、组织委托与关系债层，回答以下问题：

- 角色如何和更大的组织、势力、盟友网络绑定
- 委托系统在本作里为什么不是任务板
- 中后期关系债为什么不能沿用首发的轻量债务语义

## 实现前公共接口问题

- 该系统的主循环是什么，最小操作单位是什么
- 它消耗哪些：时间、AP、资源、暴露、债务、人情、空间占用
- 它产出哪些：成长、材料、资格、部署能力、交易能力、关系变化
- 它依赖哪些上位状态：修为、空间、节点、路线、Build、资产所有权
- 它的失败形态是什么：受损、拖慢、欠账、暴露、路线断裂、资产损失
- 它在 `canon_strict` 与 `sandbox_if` 下哪里不同
- 它要在账本 UI 上最少显示哪些信息
- 它如何与首发系统兼容，而不是推翻首发系统

## 设计目标

### 1. 势力依附必须是真支撑，也是真束缚

- 依附不是单向 buff
- 你拿到的支持必须换来义务、限制、风险或站队后果

### 2. 委托不是任务板

- 委托必须带背景、代价、履约后果和关系牵连
- 玩家接的不是日常任务，而是对关系网和资源链的承诺

### 3. 中后期关系债必须升维

- 首发是药堂债、人情债、旁支压力债
- 中后期则扩展为组织债、盟约债、供给债、信誉债

## 势力依附定位

中后期势力依附至少回答：

- 你归于谁
- 你受谁保护
- 你受谁约束
- 你能调用谁的资源
- 你会为谁背锅

## 组织委托主循环

建议按以下循环组织：

1. 接触或维持组织关系
2. 看见委托入口
3. 评估收益、代价、履约能力和牵连
4. 接受、拖延、转包或拒绝
5. 结算结果与关系后果

最小操作单位建议为 `一次委托承诺`。

## 委托输入

- 当前势力归属
- 当前 Build 与可交付能力
- 当前空间可达性
- 当前资源与供给链
- 当前信誉与既有债务

## 委托产出

委托至少可能产出：

- 新资格
- 新资源源头
- 新交易渠道
- 新部署需求
- 更深绑定关系

## 委托失败形态

常见失败形态：

- 信誉受损
- 供给链断掉
- 被迫补偿
- 组织支持缩水
- 被卷入更深站队

## 关系债分层

中后期关系债建议至少区分：

- 组织债
- 盟约债
- 供给债
- 信誉债

它们都不同于首发的药堂债。

## 与首发债务的延续

首发债务不是作废，而是演化素材。

例如：

- 首发习惯靠谁救命，会影响你中后期更容易投向哪类组织
- 首发偏好的门路和灰线，会影响你中后期更容易欠哪种关系债

## 与交易和部署的关系

势力依附将直接影响：

- 你能否调货
- 你能否托管资产
- 你能否借阵或仙蛊屋部件
- 你有没有稳定委托来源

## 模式差异

### canon_strict

- 组织边界与站队后果更硬
- 委托条件与势力关系更受原著逻辑约束

### sandbox_if

- 允许更激进站队
- 允许更早切换组织或走灰色依附

## 账本 UI 最低要求

相关页至少要显示：

- 当前主要依附对象
- 当前最重要委托
- 当前组织债或信誉债
- 当前可调用与不可调用的支持

## 验收标准

- 势力依附被定义为支持与束缚并存
- 委托不是纯任务板
- 中后期关系债与首发债务边界清楚
- 关系债真实影响交易、空间和部署

## 失败信号

- 组织只是 buff 来源
- 委托像 MMO 日常
- 关系债没有长期牵连
- 首发债务与中后期信誉债混成一个数字
