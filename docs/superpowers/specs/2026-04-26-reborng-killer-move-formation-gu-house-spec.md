## 文档角色

本文档负责定义杀招、阵与仙蛊屋在中后期的 Build 和部署语义，回答以下问题：

- 杀招是什么，和首发“决断动作”如何连接
- 阵是什么，为什么它不是普通技能
- 仙蛊屋是什么，为什么它必须当成资产与部署载体处理
- 三者如何共用 Build 底座，而不另起角色系统

## 实现前公共接口问题

- 该系统的主循环是什么，最小操作单位是什么
- 它消耗哪些：时间、AP、资源、暴露、债务、人情、空间占用
- 它产出哪些：成长、材料、资格、部署能力、交易能力、关系变化
- 它依赖哪些上位状态：修为、空间、节点、路线、Build、资产所有权
- 它的失败形态是什么：受损、拖慢、欠账、暴露、路线断裂、资产损失
- 它在 `canon_strict` 与 `sandbox_if` 下哪里不同
- 它要在账本 UI 上最少显示哪些信息
- 它如何与首发系统兼容，而不是推翻首发系统

## 设计目标

### 1. 杀招、阵、仙蛊屋是 Build 高阶层

- 它们不是新职业
- 它们是 Build 在战斗、场域和载体上的高级展开

### 2. 场域和部署必须进入系统核心

- 阵和仙蛊屋要求玩家思考位置、准备、维护和部件
- 不能只做成面板加成词条

### 3. 三者必须共享资产与维护逻辑

- 杀招吃核心条件
- 阵吃场域与布置条件
- 仙蛊屋吃部件、维护与资产占位

## 杀招定位

杀招是高阶 `决断动作模板`。

它至少由以下部分组成：

- 核心前提
- 所需蛊或资源
- 激活条件
- 效果方向
- 代价与冷后果

## 阵定位

阵是 `场域部署结构`。

它至少由以下部分组成：

- 布置条件
- 占位需求
- 维持需求
- 覆盖场域
- 失效条件

## 仙蛊屋定位

仙蛊屋是 `组合型外部载体资产`。

它至少由以下部分组成：

- 部件位
- 组合关系
- 激活方式
- 维护成本
- 使用场景

仙蛊屋不应被视为一张普通装备卡。

## 主循环

### 杀招

1. 形成模板
2. 满足条件
3. 在冲突中择机使用
4. 结算代价与后果

### 阵

1. 准备场域
2. 投入布置
3. 保持运转
4. 被打断、完成或回收

### 仙蛊屋

1. 组装或维护
2. 决定何时投入使用
3. 在合适空间中部署
4. 结算损耗、价值和后续维护压力

## 依赖关系

三者都依赖：

- 修为阶段
- Build 主轴
- 资产容器
- 空间层
- 经营支撑
- 交易或组织供给

## 失败形态

### 杀招失败

- 资源白耗
- Build 暴露
- 进入更差局面

### 阵失败

- 场域被破
- 投入白费
- 被迫换打法

### 仙蛊屋失败

- 部件受损
- 维护链断裂
- 大资产沉没

## 与首发系统的兼容

- 首发的决断动作是杀招的低阶祖型
- 首发的节点与路线风险是阵与仙蛊屋未来的场域基础
- 首发的 Build 结构是三者的前置母体

## 资产与所有权

三者都必须进入资产容器逻辑：

- 哪些是随身可用
- 哪些在仙窍内
- 哪些在部署中
- 哪些在维护中
- 哪些在交易占用中

## 模式差异

### canon_strict

- 组合更受原著与证据约束
- 过强联动更谨慎

### sandbox_if

- 允许更大胆的原创组合
- 允许更偏门的部署和载体思路

## 账本 UI 最低要求

相关页至少要显示：

- 当前核心杀招模板
- 当前阵位或布置状态
- 当前仙蛊屋部件与维护状态
- 当前依赖的资源与容器

## 验收标准

- 三者都被定义为 Build 高阶层
- 阵与仙蛊屋有空间和资产语义
- 杀招与首发决断动作逻辑可衔接
- 使用与维护代价清楚

## 失败信号

- 杀招只是普通技能树
- 阵只是被动 buff
- 仙蛊屋只是高阶装备栏
- 三者与资产容器和经营支撑脱钩
