## 文档角色

本文档负责定义从首发小地图扩展到跨图与地区切换的世界空间逻辑，回答以下问题：

- 角色如何从青茅山离开并进入更大世界
- 多地区空间如何组织，而不压垮首发地图模型
- 公开地图、边缘接触点、跨图区、仙窍与交易空间如何共存

## 实现前公共接口问题

- 该系统的主循环是什么，最小操作单位是什么
- 它消耗哪些：时间、AP、资源、暴露、债务、人情、空间占用
- 它产出哪些：成长、材料、资格、部署能力、交易能力、关系变化
- 它依赖哪些上位状态：修为、空间、节点、路线、Build、资产所有权
- 它的失败形态是什么：受损、拖慢、欠账、暴露、路线断裂、资产损失
- 它在 `canon_strict` 与 `sandbox_if` 下哪里不同
- 它要在账本 UI 上最少显示哪些信息
- 它如何与首发系统兼容，而不是推翻首发系统

## 设计目标

### 1. 不把世界写成单一 currentRegion

- 世界空间必须分层
- 不能用一个“当前地区”字段概括全部位置语义

### 2. 保留首发地图逻辑

- 首发节点地图仍然有效
- 但要成为更大世界空间中的局部地图，而不是被废弃

### 3. 切图必须有代价

- 跨图不是免费瞬移
- 应带来时间、风险、资格、关系或资源代价

## 世界空间层

世界空间至少分为：

- 局部节点图
- 地区层
- 跨图区
- 仙窍内空间
- 交易空间

## 局部节点图

- 保留首发的节点/路线/暴露逻辑
- 每个地区内部都可以有局部节点图

## 地区层

- 用来组织青茅山、城郊、北原、中央大陆等起点和中后期区域
- 地区层不是节点列表，而是更高一层的活动舞台

## 跨图区

- 用来表达地区之间的旅行、转运、过境和转换风险
- 它不等于局部地图放大版

## 地区切换主循环

建议按以下循环组织：

1. 决定是否离开当前地区
2. 判断资格、资源与风险是否允许
3. 选择路线或门路
4. 结算跨图区代价
5. 到达新地区并接入新局部地图

最小操作单位建议为 `地区转移决策`。

## 地区切换输入

- 当前修为阶段
- 当前地区身份与势力关系
- 当前资源层状态
- 当前暴露与追索
- 当前 Build 所需资源或门路
- 当前是否有安全落脚或引路人

## 地区切换代价

常见代价应包括：

- 长时间损耗
- 资源消耗
- 暴露或追索变化
- 某些关系断开或减弱
- 某些债务无法轻易拖住

## 地区切换产出

切换地区后常见收益包括：

- 新资源层机会
- 新 Build 支撑
- 新组织关系
- 新黑市或交易门路
- 脱离旧秩序束缚

## 首发与中后期衔接

首发只需要把“局部节点图是世界空间的局部一层”写清，不需要马上开放大地图旅行。

中后期再逐步开放：

- 地区离开资格
- 稳定跨区门路
- 更大的落脚与依附逻辑

## 模式差异

### canon_strict

- 地区切换更受原著地理与时序约束
- 不轻易提前开放不合理地区

### sandbox_if

- 允许更早偏航
- 允许更大胆地通过灰路、组织或特殊门路离开原区域

## 账本 UI 最低要求

世界空间页至少要显示：

- 当前局部地图属于哪个地区
- 当前可触达的下一层空间
- 当前跨图所需的主要代价
- 当前地区外的主要门路或阻碍

## 验收标准

- 首发小地图被明确定义为更大空间层的一部分
- 地区切换不是免费按钮
- 主地图、仙窍、交易空间不会被写成同一种空间

## 失败信号

- 一切都塞进一个 currentRegion 概念
- 切图没有实际代价
- 首发地图一到中后期就失效重做
